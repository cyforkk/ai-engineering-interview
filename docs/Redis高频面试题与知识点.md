## 二、P0 必会

### 2.1 为什么快

1. 内存  
2. 单线程执行命令（无锁竞争、少切换）  
3. 高效数据结构  
4. IO 多路复用  

（Redis6 IO 多线程≠命令多线程执行，面试说清口径。）

### 2.2 五类型（场景）

| 类型 | 场景 |
|------|------|
| String | 缓存、计数、Session、锁 |
| Hash | 对象字段 |
| List | 最新列表、简易队列 |
| Set | 去重、共同关注 |
| ZSet | **排行榜**、延时（score） |

扩展：Bitmap / HLL / Stream（加分）。

**高频题：** 五类型场景？ZSet 底层（跳表+字典）？

---

### 2.3 缓存三连（第一优先级场景）

| 问题 | 定义 | 方案 |
|------|------|------|
| **穿透** | 查不存在 → 打 DB | 空值缓存、布隆 |
| **击穿** | 热点过期瞬间 | 互斥重建、逻辑过期 |
| **雪崩** | 大量同时过期/Redis 挂 | TTL 打散、高可用、限流、多级缓存 |

**高频题：** 三者区别？各怎么解？

**口述：**  
> 穿透没有、击穿热点、雪崩一片；空值/布隆、互斥或逻辑过期、打散+集群+限流。

---

### 2.4 分布式锁（P0）

```text
SET key unique NX EX seconds
解锁：Lua 校验 unique 再 DEL
```

| 点 | 结论 |
|----|------|
| NX+EX | 原子加锁防死锁 |
| unique | 防误删别人锁 |
| 续期 | Redisson 看门狗 |
| 风险 | 主从切换可能丢锁（说清可接受度） |

**高频题：** 手写锁要点？为何 Lua 删？和 Redisson？

## 三、P1 常考

### 3.1 与 DB 一致性

- 常用：**先更 DB，再删缓存**（Cache Aside）  
- 删失败：重试 / binlog 补偿  
- 最终一致；强一致成本高  

### 3.2 过期与淘汰

- 删除：惰性 + 定期  
- 满内存：`allkeys-lru` 等 maxmemory-policy  

### 3.3 持久化

| | RDB | AOF |
|--|-----|-----|
| 内容 | 快照 | 写命令 |
| 恢复 | 快 | 更全可重写 |
| 丢失 | 快照间隔 | 看 fsync |

生产常混合。

### 3.4 大 key / 热 key

拆分、压缩；本地缓存/打散/限流。
