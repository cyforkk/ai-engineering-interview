## 二、为什么快 + 数据类型（P0）

### 白话

1. **内存**计算，比磁盘 DB 快几个数量级  
2. **命令执行单线程**（逻辑简单，无锁竞争；Redis 6+ 多线程主要加速网络读写，别说成「多线程随便并行执行命令」）  
3. 高效编码与数据结构  
4. **IO 多路复用** 处理大量连接  

| 类型 | 典型场景 |
|------|----------|
| String | 缓存对象 JSON、计数器、Session、分布式锁 |
| Hash | 对象多字段（用户资料局部更新） |
| List | 最新消息列表、简单队列 |
| Set | 去重、共同关注 |
| ZSet | **排行榜**、延时任务（score 当时间） |

### 面试怎么答（约 1 分钟）

> Redis 快主要因为内存、单线程执行命令减少锁、高效数据结构和 IO 多路复用。  
> 五类型我会各举场景：String 做缓存和锁，Hash 做字段级对象，List 做动态列表，Set 做关系去重，ZSet 做排行榜。  
> 选型先看访问模式，不要所有东西都塞 String。

## 三、缓存三连（P0，最易混）

必须先 **定义差异**，再讲方案——混成一套话会被追问穿。

| 问题 | 一句话定义 | 核心方案 |
|------|------------|----------|
| **穿透** | 查 **根本不存在** 的数据，缓存不中，每次打到 DB | 缓存空值、布隆过滤器、参数校验 |
| **击穿** | **热点 key** 过期瞬间，大量并发重建 | 互斥锁重建、逻辑过期、永不过期+异步刷新 |
| **雪崩** | **大量 key 同时过期** 或 Redis 整体不可用 | TTL 加随机抖动、集群高可用、限流降级、多级缓存 |

### 面试怎么答（约 2 分钟，建议背结构）

> 我先区分三个概念。  
> 穿透是恶意或乱查不存在的 key，缓存永远帮不上忙，用布隆或空值缓存挡住。  
> 击穿是单个热点突然失效，用互斥只让一个线程回源，或逻辑过期避免真的删掉。  
> 雪崩是大面积一起失效或 Redis 挂了，TTL 打散，加上集群和本地缓存、接口限流。  
> 三者方案不能混用一套词，面试官会故意追问差异。

**完整长答：** [Redis面渣 · 三连](./Redis面渣级口述.md)

### 追问

| 问 | 答 |
|----|-----|
| 空值缓存会占内存？ | 短 TTL；布隆更省但有误判 |
| 逻辑过期是什么？ | value 里带过期时间，过期仍返回旧值并异步重建 |

## 四、分布式锁（P0）

### 白话

多机部署时，JVM 的 `synchronized` 只锁本机。要用 Redis：

```text
SET lock_key unique_value NX EX 30
```

- **NX**：不存在才成功 → 互斥  
- **EX**：超时自动释放 → 防死锁（持锁进程挂了）  
- **unique_value**：删除时确认是自己的锁，避免误删别人的  
- 删除用 **Lua**：`if get==me then del` 保证原子  

业务执行时间可能超过 EX → 需要 **续期**（Redisson 看门狗）。  
Redis 主从异步复制时，主挂可能丢锁 → 讲清业务能否接受；强需求可看 RedLock 争议或 ZooKeeper/etcd。

### 面试怎么答（约 1.5 分钟）

> 分布式锁我用 SET key value NX EX，保证互斥和超时。  
> value 用唯一 ID，释放时用 Lua 校验再删，防止超时后误删别人的锁。  
> 长任务要续期。  
> 同时说明：锁超时要大于业务耗时或可续期；Redis 故障转移在极端情况下可能有安全隐患，金融级要评估。

**完整稿：** [Redis面渣 · 锁](./Redis面渣级口述.md)

## 五、缓存与 DB 一致性（P1）

### 白话

常见 **Cache Aside**：

1. 读：先缓存，没有再 DB，再回填缓存  
2. 写：**先更新 DB，再删除缓存**（不是先改缓存）

删缓存失败 → 重试、或基于 binlog 的异步删除补偿。  
绝对强一致很难，互联网业务多接受 **最终一致**，用 TTL 兜底。

### 面试怎么答

> 读用旁路缓存；写先 DB 再删缓存，避免先改缓存导致脏数据更难修。  
> 删失败要补偿。强一致要上分布式事务或换架构，成本和复杂度高。

### 追问：为何不先更新缓存？

> 并发下更容易出现「缓存新、DB 旧」或写覆盖顺序问题；删缓存让下次读回源更稳妥（仍非绝对完美，但是主流实践）。
