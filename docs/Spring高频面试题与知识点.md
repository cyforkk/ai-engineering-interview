## 1. IOC / DI？

- **IOC**：控制反转，对象创建与装配交给容器。
- **DI**：依赖注入（构造器 / setter / 字段），容器把依赖塞进来。
- 好处：解耦、易测、易替换实现。

---

## 2. AOP？代理？

- 面向切面：日志、事务、权限等横切逻辑。
- 实现：动态代理
  - 有接口：JDK 动态代理。
  - 无接口：CGLIB 子类代理。
- **自调用**（`this.xxx()`）不走代理 → 事务等注解可能失效。

---

## 3. @Transactional 失效常见原因？

- 同类内部调用。
- 方法非 public（视配置）。
- 异常被 catch 吞掉未抛出。
- 异常类型/传播行为配置不当。

---

## 4. Bean 作用域？循环依赖？

- 默认**单例**；还有 prototype、request、session 等。
- 单例循环依赖：三级缓存可解部分（字段/setter）；**构造器循环**通常无法解。

---

## 5. Spring Boot 自动配置？

- 条件注解 + starter + 自动配置类。
- classpath / 配置满足则自动装配 Bean。
- 出问题可查自动配置报告。
