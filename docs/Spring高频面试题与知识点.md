## 二、IoC / DI / AOP（P0）

### 白话

**没有 IoC：** 你在代码里 `new` 依赖，类和类死绑在一起，难测难换。  
**IoC：** 对象交给 Spring 容器创建；**DI** 再把依赖注入进来（推荐构造器注入）。  

**AOP：** 日志、事务、鉴权这些横切逻辑，不想在每个方法复制粘贴，就用代理在方法前后织入。  
Spring 事务、统计耗时，都是 AOP。

### 面试怎么答

> IoC 控制反转，DI 注入依赖，解耦易测。  
> AOP 横切，动态代理实现；自调用不走代理。

## 三、事务失效（P0 必背清单）

### 白话

事务靠 **AOP 代理** 包一层：进方法前开事务，正常提交，异常回滚。  
如果请求**没走到代理**，注解等于废纸。

**常见失效（按出现频率说）：**

1. **同类自调用** `this.b()` — 最经典  
2. 方法不是 public  
3. 异常被 catch 吃掉  
4. 受检异常未 `rollbackFor`  
5. 引擎不支持事务 / 不是 Spring Bean  
6. 传播行为配错  
7. 异步子线程无事务上下文  

### 面试怎么答

> 原理代理 + 失效场景至少讲 4 条 + 怎么修（拆 Bean、注入自己、抛出异常、rollbackFor）。  
> REQUIRED vs REQUIRES_NEW：加入当前 vs 挂起并新开。

**完整稿：** [Spring面渣 · 事务长答](./Spring面渣级口述.md)

## 四、三级缓存与循环依赖（P0）

### 白话

A 依赖 B，B 依赖 A。Spring 单例字段注入时：  
A 先实例化成「半成品」放进缓存 → 去造 B → B 需要 A 时拿到半成品 → B 完成 → A 再注 B 完成。  

**三级缓存**是为了可能提前暴露**代理对象**。  
**构造器循环**解不了（创建时就要对方完整实例）。

### 面试怎么答

> 单例+setter/字段可解；构造器不行；三级各自作用；设计上应少循环依赖。

**完整稿：** [Spring面渣 · 三级缓存](./Spring面渣级口述.md)

## 五、生命周期 / MVC / Boot（P1）

**生命周期口诀：** 实例化 → 属性填充 → 初始化（PostConstruct 等）→ 后置处理（常在这生成 AOP 代理）→ 使用 → 销毁。

**MVC：** DispatcherServlet 前端控制器 → 映射 Handler → 适配器执行 → 返回 JSON/视图。

**Boot 自动配置：** starter 带依赖 + 条件装配（classpath 有某类才配 Bean），约定优于配置。
