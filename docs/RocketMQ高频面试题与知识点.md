## 和通用 MQ 题的关系

| 通用考点（完整卷） | RocketMQ 上怎么落地 |
|--------------------|---------------------|
| 不丢 | 同步发送/刷盘与多副本策略；消费成功再 ACK |
| 不重 | 业务幂等 + 唯一键（至少一次） |
| 顺序 | 顺序消息 / 同队列顺序消费（吞吐换有序） |
| 积压 | 扩消费者、优化消费、临时主题分流 |
| 事务一致性 | **事务消息**：半消息 + 本地事务 + 回查 |

---

## RocketMQ diff（只记这些）

| 点 | 一句话 |
|----|--------|
| NameServer | 轻量路由注册，无强依赖 ZK |
| Broker | 存消息；主从复制 |
| 集群 / 广播消费 | 集群分摊；广播每人一份 |
| 事务消息 | 半消息 → 本地事务 → 提交/回滚；Broker 回查 |
| 延时消息 | 定时/延时投递（注意级别与版本能力） |
| 死信 | 多次失败进死信，需人工/补偿 |

## 选型一句

- **电商订单、事务消息、延时取消** → 国内业务常选 RocketMQ  
- 超高吞吐日志流 → 常对比 Kafka（见完整卷三大对比）
