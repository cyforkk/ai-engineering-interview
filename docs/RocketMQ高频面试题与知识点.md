## 二、核心模型 P0

| 组件 | 作用 |
|------|------|
| NameServer | 路由发现（轻量） |
| Broker | 存消息 |
| Producer / Consumer | 生产消费 |
| Topic / Tag | 主题与过滤标签 |
| Consumer Group | 消费组 |

### 消费模式

| 模式 | 说明 |
|------|------|
| 集群 | 组内分摊（默认业务） |
| 广播 | 组内每人一份 |

### 可靠相关

| 点 | 说明 |
|----|------|
| 同步发送 | 要确认 |
| 刷盘 | 同步/异步刷盘权衡 |
| 复制 | 同步双写更安全 |
| 消费重试 | 失败进重试队列 |
| 死信 | 多次失败进 DLQ |
| 事务消息 | 半消息 + 本地事务反查 |

### 顺序消息

- 分区顺序：相同 orderId 进同一队列  
- 消费端单线程该队列  

### 延时消息

- 定时/延时级别（版本支持不同）  
- 关单场景常用

## 三、事务消息流程（必背）

```text
1. 发半消息
2. 执行本地事务
3. 成功 Commit / 失败 Rollback
4. 未知则 Broker 反查生产者
5. 消费者只看到 Commit 后的消息
```

与本地消息表对比：半消息少一张业务表，但要反查接口。
