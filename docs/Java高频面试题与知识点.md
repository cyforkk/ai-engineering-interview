## 二、equals / hashCode / ==

### 白话

- `==`：基本类型比「值一不一样」；引用类型比「是不是同一个对象」（地址）。  
- `equals`：Object 默认也是比地址；String、Integer 等重写了，变成比「内容」。  
- 你自己的类如果按业务含义相等（两个用户 id 相同算同一人），就要自己写 equals。  
- **hashCode**：把对象映射成一个 int，给 HashMap 这类结构「先粗分桶」用。

**契约（必须背熟含义）：**  
两个对象 equals 为 true，hashCode **必须**一样。  
否则 HashMap 会：先按 hash 进 A 桶，equals 却认为和 B 桶里的相等——逻辑就乱了，put 进去 get 不回来。

### 面试怎么答（约 1 分钟）

> == 比基本类型值或引用地址；equals 默认同地址，可重写成比内容。  
> 重写 equals 必须重写 hashCode，保证相等对象 hash 相同。  
> HashMap 先用 hash 找桶，再用 equals 比 key，所以契约很重要。

### 追问

| 问 | 答 |
|----|-----|
| 只重写 equals 不重写 hashCode？ | HashMap/HashSet 行为错乱 |
| hash 相同一定 equals？ | 不一定，只是可能碰撞 |
| 记录类 record？ | 自动生成较规范的 equals/hashCode |

**完整口语稿：** [Java面渣 · §1](./Java面渣级口述.md)

## 三、String 为什么不可变

### 白话

String 一旦创建，字符内容不能改。类是 final，内部数组不暴露修改方法。  
你以为的 `s = s + "a"` 其实是**新对象**，不是在原字符串上改。

**为什么这么设计（说得出 3 点就够）：**

1. **常量池可共享**：相同字面量复用，省内存，且安全。  
2. **可缓存 hashCode**：做 HashMap 的 key 很快。  
3. **线程安全、防篡改**：类名、路径、网络参数不易被中途改掉。

拼接：循环里用 `StringBuilder`；`StringBuffer` 带锁，现代代码少用。

### 面试怎么答

> String 不可变：final 类 + 无修改内容的 API。  
> 好处是池化共享、hash 可缓存、天然线程安全。  
> 频繁拼接用 StringBuilder。

### 追问

| 问 | 答 |
|----|-----|
| `new String("a")` 几个对象？ | 字面量池里可能已有；new 一定在堆再造一个 |
| intern？ | 尝试放入/返回池中引用 |

**完整口语稿：** [Java面渣 · String](./Java面渣级口述.md)

## 四、HashMap（最重要）

### 白话

可以想成：**一排抽屉（数组）**，每个抽屉里可能挂一条链，链太长改成红黑树，查找更快。  
放 key 时：先算 hash → 决定进哪个抽屉 → 抽屉里用 equals 找有没有相同 key。

**JDK8 默认：** 容量 16，负载因子 0.75（装到 75% 就扩容成 2 倍）。  
**树化：** 某桶链表长度 ≥8 且数组长度 ≥64 才变红黑树；树很小（≤6）再变回链表。  
**线程不安全：** 多线程一起 put 可能丢数据；并发用 ConcurrentHashMap。

### 面试怎么答（必须能说 2 分钟）

建议结构：

1. 结构：数组 + 链表 + 红黑树  
2. put：hash → 下标 → 空则放 / 相等则覆盖 / 链上追加 → 可能树化/扩容  
3. 为何 2 的幂：`(n-1)&hash` 代替取模  
4. 扩容：元素要么在原下标，要么在「原下标+旧容量」  
5. 非线程安全 → CHM  

### 追问（高频）

| 问 | 答 |
|----|-----|
| 为何树化阈值 8？ | 链长短时链表更省；泊松分布下很长很少；8 是折中 |
| 为何还要数组≥64？ | 太短优先扩容摊平，而不是急着树化 |
| 多线程会怎样？ | 数据错乱；老版本还有死链传说，JDK8 改善但仍不安全 |
| key 要怎样？ | 正确 equals/hashCode，最好不可变 |

**完整口语稿（主教材）：** [Java面渣 · HashMap 长答](./Java面渣级口述.md)

## 五、ArrayList

### 白话

底层是**可变长数组**。按下标访问很快 O(1)；中间插入删除要搬元素，慢。  
空间不够大约按 1.5 倍扩容，要数组拷贝。  
foreach 里直接 list.remove 会 fail-fast 抛异常，应用迭代器的 remove。

### 面试怎么答

> ArrayList 数组实现，随机访问快，中间改动慢，约 1.5 倍扩容。  
> 非线程安全。和 LinkedList 比，一般更常用 ArrayList。

**口述：** [Java面渣](./Java面渣级口述.md)

## 六、ConcurrentHashMap（P1）

### 白话

并发版 Map。JDK8：桶空用 CAS 挂头节点；有数据时锁**桶头**（synchronized），粒度比整表锁细。  
不允许 null key/value（并发下难区分「没有」和「就是 null」）。  
get 基本不锁，靠可见性语义。

### 面试怎么答

> JDK8 CAS + 同步桶头；比 Hashtable 整表锁并发好；比 HashMap 安全。  
> 禁 null；size 是近似统计。

**详述：** [Java面渣 · CHM](./Java面渣级口述.md) · [并发专题](./并发高频面试题与知识点.md)

## 七、值传递、异常（P0 简）

### 白话

Java **只有值传递**：传对象时传的是**引用的副本**。方法里 `a = new X()` 改不了外面的指向，但 `a.setY()` 能改同一对象内容。

异常：Error 多严重（OOM）；Exception 里运行时可不声明，受检要处理。资源用 try-with-resources。
