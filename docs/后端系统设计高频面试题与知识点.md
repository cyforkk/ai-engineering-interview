> 中高级后端核心考察：**高并发 / 高可用 / 可扩展** 的设计与权衡，不是唯一标准答案。

### 标准回答框架（强烈推荐）

```text
1. 需求澄清（功能 + 非功能：QPS、数据量、一致性、可用性）
2. 容量粗算
3. 整体架构（接入 → 网关 → 服务 → 缓存 → DB → MQ）
4. 关键技术点深入
5. 瓶颈与优化
6. 兜底与监控
```

每个方案要能说：**为什么选 · 缺点 · 如何兜底**。大厂重 **trade-off**。

### 专项时间占比

| 优先级 | 模块 | 时间 |
|--------|------|:----:|
| **P0** | 秒杀 + 短链 + 分布式 ID + 限流 | **45%** |
| **P1** | 订单超时 + 点赞计数 + 海量 UV/去重 | **25%** |
| **P2** | 缓存一致 + 排行榜 + 大文件 | **15%** |
| **P3** | RPC、微服务、分布式事务等 | **15%** |

# 一、超高频（几乎必问）

## 1. 秒杀系统

### 1.1 整体思路（10 万抢 100 件）

| 层 | 手段 |
|----|------|
| 前端 | 按钮防抖、答题/验证码、静态化、CDN |
| 网关 | 限流、黑名单、WAF |
| 服务 | 库存预扣、排队、异步下单 |
| 缓存 | Redis 预热库存、Lua 原子扣减 |
| DB | 少写；最终落库；乐观锁/条件更新 |
| MQ | 削峰异步创建订单 |

```text
流量漏斗：拦截无效 → 限流 → Redis 扣库存 → MQ → 异步下单落库
```

### 1.2 如何防超卖？

- Redis `DECR` / Lua：库存 >0 才扣  
- DB：`UPDATE stock SET num=num-1 WHERE num>0 AND id=?`  
- 订单唯一、状态机；对账补偿  

### 1.3 瞬时高并发？

- **限流**（网关+接口）  
- **削峰**（MQ）  
- **异步**下单  
- 缓存挡读、预热  

### 1.4 库存 Redis 还是 DB？一致性？

- 热点库存：**Redis 预扣** 扛并发  
- 最终：**DB 为准** 落库校验  
- 不一致：定时对账、消息重试、人工/自动回补库存  

### 1.5 分层防护

| 层 | 防护 |
|----|------|
| 前端 | 限点、隐藏真实接口 |
| 网关 | 令牌桶/漏桶、IP/UID 限流 |
| 服务 | 库存预检查、幂等 token、熔断 |

## 2. 短链系统

### 2.1 架构直觉

```text
生成：长链 → 发号/哈希 → 短码 → 存映射
跳转：短码 → 查映射 → 302/301 → 长链
```

### 2.2 短码如何生成？

| 方案 | 说明 |
|------|------|
| **分布式 ID + Base62** | 趋势递增、可控制长度 |
| 哈希截断 | 简单有冲突，需冲突检测 |
| 预生成号池 | 高性能 |

### 2.3 不重复与扩展？

- 唯一索引；冲突重试  
- 分库分表 / 缓存热点短链  
- 号段/雪花保证全局唯一  

### 2.4 跳转高可用 + 统计？

- 多副本 API、缓存映射、CDN  
- 异步写点击日志/MQ 做统计（UV/PV）  
- 302 临时；301 可缓存（慎用）  

---

## 3. 分布式 ID 发号器

### 3.1 方案对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| UUID | 简单 | 无序、长、索引不友好 |
| DB 自增 | 简单 | 单点、性能瓶颈 |
| **号段模式** | 吞吐高 | 依赖 DB 号段表 |
| **雪花 Snowflake** | 趋势递增、高性能 | **时钟回拨** |
| 美团 Leaf 等 | 生产级 | 运维复杂度 |

### 3.2 雪花算法

```text
1 符号位 + 41 时间戳 + 10 机器 + 12 序列
```

- **时钟回拨**：拒绝/等待/借用历史序列/换算法  
- 保证：趋势递增、高性能、机器位规划高可用  

### 3.3 目标

- 全局唯一  
- 趋势递增（友好索引）  
- 高性能、高可用  

---

## 4. 限流与高并发通用

### 4.1 限流算法

| 算法 | 特点 |
|------|------|
| 固定窗口 | 实现简单，边界突刺 |
| 滑动窗口 | 更平滑 |
| **令牌桶** | 允许一定突发 |
| 漏桶 | 恒定流出 |

### 4.2 单机 vs 分布式

| 单机 | 分布式 |
|------|--------|
| Guava/RateLimiter、本地信号量 | Redis + Lua、网关集群限流、Sentinel |

### 4.3 QPS 暴增 10 倍？

1. 限流保护核心  
2. 扩容（水平）  
3. 降级非核心  
4. 缓存/CDN 挡读  
5. MQ 削峰  
6. 熔断依赖  

### 4.4 接口防刷

- 登录态、验证码、设备指纹  
- UID/IP 限流  
- 业务 token 一次性  
- 风控规则  

---

## 5. 点赞 / 计数

### 5.1 高并发点赞

```text
点赞写 Redis（Set/Bitmap 记用户 + String/Hash 计数）
 → 异步落库
 → 读：缓存计数 + SISMEMBER 是否已赞
```

### 5.2 实时总数 + 是否已赞

- 总数：Redis 计数  
- 是否已赞：Set/Bitmap/布隆（注意误判）  

### 5.3 热点 Key

- 本地缓存 + Redis  
- key 打散（分片计数再聚合）  
- 读写分离、限流

# 二、高频

## 6. 订单与业务

### 6.1 订单超时取消

| 方案 | 说明 |
|------|------|
| 延迟队列 | RocketMQ 延时、Rabbit TTL+死信 |
| Redis ZSet | score=过期时间，轮询 |
| 定时任务扫表 | 简单，实时性一般 |
| 状态机 | 待支付→取消，幂等 |

详见口述：[延时消息](./系统设计口述-延时消息.md) · [支付幂等](./系统设计口述-支付与幂等.md)

### 6.2 抢单不重复

- Redis 锁 / DB 条件更新 `WHERE status=待抢`  
- 唯一约束；乐观锁 version  

### 6.3 提现失败 / 退单

- 状态机 + 补偿事务  
- 幂等单号；对账

## 7. 海量数据

| 问题 | 思路 |
|------|------|
| UV/PV | PV 计数；UV 用 HyperLogLog/Bitmap |
| 40 亿去重 1G | 位图（若范围可控）/ 分片布隆 / 外部存储 |
| 推荐去重 | 布隆过滤器 + 过期重建 |
| TopK | 小顶堆、快选、分治 |

## 8. 缓存与一致性

- **Cache Aside**：先 DB 再删缓存；TTL 兜底；binlog 补偿  
- 热点：本地 + Redis；逻辑过期  
- 三连综合：空值/布隆、互斥、TTL 随机、HA、限流

# 三、中频

## 10. RPC 框架要点

- 序列化、协议、注册发现、负载均衡、超时重试、熔断  
- 同步/异步、连接池  

## 11. 动态线程池

- 配置中心改 core/max/queue  
- 监控拒绝、队列、耗时；优雅生效  

## 12. 负载均衡与一致性哈希

| 策略 | 场景 |
|------|------|
| 轮询/随机/加权 | 无状态服务 |
| 最小连接 | 长连接 |
| **一致性哈希** | 缓存节点扩缩，减少失效 |

- 虚拟节点减倾斜  

## 13. 分库分表

| 点 | 内容 |
|----|------|
| 路由 | hash、range、地理位置 |
| 问题 | 跨库 join、事务、扩容 |
| 扩容 | 双倍扩容、一致性哈希、停机迁移 |
| ID | 雪花/号段 |

## 14. 微服务拆分

- 业务边界、DDD 限界上下文  
- 高内聚低耦合；避免分布式大单体  

## 15. 分布式事务

| 方案 | 特点 |
|------|------|
| 2PC | 强一致、阻塞 |
| TCC | 业务侵入 |
| Saga | 长事务补偿 |
| 本地消息表 / 事务消息 | 最终一致常用 |
| Seata | 框架化 |

## 16. OAuth2 / 登录安全

- 授权码模式常见第三方登录  
- 验证码：频控、一次有效  
- 密码错误：次数限制 + 锁定 + 验证码
