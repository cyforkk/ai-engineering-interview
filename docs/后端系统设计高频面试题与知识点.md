## 一、通用答题框架（必背）

```text
1. 澄清：功能、量级（DAU/QPS/容量）、一致性、延迟
2. 估规模：QPS、存储、带宽（数量级）
3. API 与核心模型
4. 高阶架构图：接入-服务-存储-异步-观测
5. 深入 1～2 个难点（热点、一致、削峰）
6. 降级容灾与演进
```

## 五、短链设计提纲（示例骨架）

1. 写：`long_url → id → base62 short`；防重复  
2. 读：缓存 → DB → 302  
3. 量级：写少读多，缓存为王  
4. 安全：不可枚举、恶意链接审核  
5. 数据：冷热分离、过期链接  

---

## 六、秒杀提纲

1. 前端+网关限流  
2. 库存预减 Redis + 异步下单  
3. 幂等令牌  
4. 热点 key 打散  
5. 降级：排队、验证码
