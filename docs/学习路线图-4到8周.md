# 学习路线图 · 4 周冲刺 / 8 周扎实

> 入口：[README.md](./README.md) · 仓库 [ai-engineering-interview](https://github.com/cyforkk/ai-engineering-interview)  
> 用法：每天 **1.5～3 小时**；以 **P0 + 面渣录音** 为主，P2 可砍。

---

## 一、三条主路径（先选一条）

| 路径 | 适合 | 核心文档 |
|------|------|----------|
| **A. AI 工程** | AI 应用 / LLM 工程 | LLM基础 → Prompt工程 → AI应用 → 向量库 → 安全 → Agent → LangGraph选型 → LLMOps → 推理 → FastAPI → STAR → 行为面 |
| **B. Java 后端** | Java 服务端 | Java → 并发 → JVM → MySQL → Redis → Spring → MQ/Kafka或RMQ → 微服务 → 系统设计(Feed/搜索/延时) → K8s → STAR → 行为面 |
| **C. Python 后端/AI 服务** | Python API + AI | Python → 异步FastAPI → 手写 → 网络 → 算法 → AI应用/向量库 → K8s → STAR → 行为面 |

可 **主路径 80% + 辅路径 20%**（如 AI 主 + Java 集合不深挖）。

---

## 二、4 周冲刺（每天约 2h）

### 路径 A · AI 工程

| 周 | 主题 | 动作 |
|----|------|------|
| **W1** | 常识+应用 | LLM基础+面渣；AI应用工程 P0；Prompt工程；安全 P0；每天口述 3 题 |
| **W2** | 检索+Agent | 向量库双文件；Agent 双文件；追问三连-AI；画一张 RAG 架构 |
| **W3** | 工程化 | LLMOps；推理部署；Python异步FastAPI 长口述；Docker/K8s P0 |
| **W4** | 项目+模拟 | STAR 范例-AI 改写成自己的；系统设计短链或秒杀框架；全真模拟 2 场 |

### 路径 B · Java 后端

| 周 | 主题 | 动作 |
|----|------|------|
| **W1** | 语言+并发 | Java HashMap 面渣；并发线程池；追问-Java P0；手写 LRU/单例 |
| **W2** | 存储+JVM | MySQL 索引MVCC；Redis 三连锁；JVM 内存GC+场景 5 条 |
| **W3** | 框架+中间件 | Spring 事务三级缓存；MQ；微服务 P0；设计模式 P0 |
| **W4** | 综合 | 系统设计；Docker/K8s；STAR 范例-Java 改写；模拟面 |

### 路径 C · Python

| 周 | 主题 | 动作 |
|----|------|------|
| **W1** | 语言 | Python 知识点+面渣；手写装饰器/生成器/LRU；追问-Python |
| **W2** | 异步服务 | 异步FastAPI 全文过；网络 HTTPS；算法 P0 每天 1 题 |
| **W3** | AI 接口 | AI应用+向量库+安全 精简；LLMOps；Docker P0 |
| **W4** | 项目 | STAR；综合设计「RAG API」口述；模拟面 |

---

## 三、8 周扎实（每天约 1.5～2.5h）

在 4 周内容上 **每主题多 1 周消化 + 输出**：

| 周 | 全局建议 |
|----|----------|
| W1–W2 | 路径基础（语言/LLM 常识） |
| W3–W4 | 核心存储或 RAG/向量 |
| W5–W6 | 框架/Agent/中间件/部署 |
| W7 | 系统设计 + K8s + 可观测 |
| W8 | 项目故事打磨 + 每天模拟 1 面 + 查漏 |

**每周输出物（选 1）：**

- 1 篇自己的 STAR 终稿  
- 1 张架构图（手绘拍照）  
- 错题本 10 条口述卡  

---

## 四、每日节奏（固定）

```text
25min  知识点 P0
25min  面渣录音（不看稿说第二遍）
20min  追问三连 或 手写 1 题
10min  复盘笔记三行
周末   模拟面 45～60min 或 设计题 1 道
```
