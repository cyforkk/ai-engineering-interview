## 二、P0 必会

### 2.1 岗位在做什么

| 不是 | 是 |
|------|-----|
| 只调 ChatGPT 玩玩 | 把模型能力产品化、服务化 |
| 只追论文 SOTA | 稳定、成本、延迟、安全、可观测 |
| 只写 Prompt | 数据、检索、评测、工程闭环 |

**能力模型：** 产品理解 · LLM/RAG/Agent · 后端服务（Python/Java）· 数据与评测 · 运维观测  

### 2.2 典型架构

```text
客户端 → API 网关（鉴权/限流）→ 应用服务
         →（可选）Agent 编排
         → RAG：解析-分块-向量库-检索-重排
         → LLM Provider / 本地模型
         → 缓存 / DB / MQ / 对象存储
         → 日志·Trace·评测集
```

### 2.3 RAG（必背）

| 步骤 | 要点 |
|------|------|
| 解析 | PDF/HTML/表格质量决定上限 |
| 分块 | 大小、重叠、按标题/语义 |
| 嵌入 | 模型与维度、多语言 |
| 索引 | 向量库 + 可加关键词 |
| 检索 | topK、阈值、混合检索 |
| 重排 | cross-encoder / 业务规则 |
| 生成 | 带引用、上下文窗口管理 |
| 拒答 | 低相关则不硬答 |

**高频题：** RAG 解决什么？幻觉怎么办？分块怎么做？

### 2.4 幻觉与质量

| 手段 | 说明 |
|------|------|
| 检索 grounding | 答案必须可追溯引用 |
| 降温度 / 约束格式 | JSON Schema 等 |
| 拒答策略 | 无依据明确说不知道 |
| 人机抽检 | 关键业务 |
| 评测集回归 | 改 Prompt/模型防回退 |

### 2.5 评测（P0）

| 维度 | 例子 |
|------|------|
| 检索 | Recall@K、命中率 |
| 生成 | 忠实度、相关性、有用性 |
| 任务 | 任务完成率、人工分 |
| 工程 | 延迟 P99、费用、错误率 |
| 方法 | 金标集、LLM-as-Judge、人工 |

**RAGAS 思路：** faithfulness、answer relevance、context 相关（知概念即可）。

### 2.6 服务化可靠性

| 点 | 实践 |
|----|------|
| 超时 | 模型/向量库分别设超时 |
| 重试 | 有限次+退避；写操作幂等 |
| 限流 | 网关+租户配额 |
| 熔断 | Provider 挂了快速失败 |
| 流式 | SSE，注意断连与取消 |
| 削峰 | 队列，避免同步打爆 |
| 密钥 | 环境变量/密钥管理，不进库 |
| 观测 | request_id、token 用量、阶段耗时 |

## 三、P1 常考

### 3.1 向量库与检索增强

- 乘积量化、过滤（metadata filter）  
- 混合：向量 + BM25  
- 多路召回再融合  
- 更新：增量索引、软删除  

### 3.2 Prompt 与结构化

- 系统提示 vs 用户；角色分离  
- Few-shot、思维链（看场景）  
- 工具调用 / Function Calling  
- 输出 JSON 校验失败重试  

### 3.3 缓存

| 层 | 内容 |
|----|------|
| 完全相同问题 | 答案缓存 |
| 检索 | 查询向量/结果缓存 |
| 语义缓存 | 相似问命中（注意错误缓存） |

### 3.4 Agent 边界

- 步骤固定 → Workflow/RAG 链  
- 步骤动态、要工具 → Agent  
- 详见 Agent 专题：ReAct、防循环、HITL  

### 3.5 成本

- 模型路由：简单小模型、复杂大模型  
- 压缩上下文、摘要历史  
- 缓存命中率  
- 批处理离线任务

## 四、P2 加分

| 点 | 一句话 |
|----|--------|
| 微调 | 风格/格式/领域，不替代知识更新（RAG） |
| RAG vs 微调 | 知识常变更→RAG；行为稳定→微调可叠加 |
| 私有化 | 数据安全、硬件、运维 |
| 多模态 | 图文解析进同一流水线 |
| 安全 | 提示注入、越权、数据泄露、输出过滤 |

## 五、系统设计题框架

```text
1. 场景与指标（准确率/延迟/成本/安全）
2. 架构图（接入-编排-RAG-模型-数据-观测）
3. 数据与索引更新
4. 评测与灰度
5. 失败降级（检索失败/模型超时）
6. 演进（先 RAG 再 Agent）
```

**常见题：** 企业知识库问答、客服助手、文档审阅、代码助手、智能体调研。
