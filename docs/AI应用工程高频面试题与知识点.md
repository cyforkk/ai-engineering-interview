## 一、为什么要「应用工程」而不只是调 API

直接把前端接到模型：无法限流、无法审计、无法换模型、无法评测回归、无法处理超时与降级。  
工程化是把 LLM 当成**不可靠的高延迟依赖服务**来做架构。

---

## 二、推荐链路及每段职责

```text
客户端 → 接入层(鉴权/限流/配额)
      → 编排(工作流/Agent/RAG)
      → 模型网关(路由/超时/重试/降级)
      → 后处理(格式/安全/引用)
      → 观测(trace/log/metrics) + 计费
```

| 段 | 所以然 |
|----|--------|
| 接入 | 防止被刷爆账单与下游 |
| 编排 | 业务步骤可测可改 |
| 网关 | 模型可替换；统一超时 |
| 后处理 | 模型输出不可直接信 |
| 观测 | 没有数据就无法优化与追责 |

---

## 三、流式、超时、幂等

- **流式：** 改善 TTFT 体感；注意取消与半截输出处理  
- **超时：** 分层（网关 < 上游预算）；超时要有降级文案或缓存答案  
- **幂等：** 客户端重试常见；防止重复扣费、重复写业务
