## 一、要搞定什么

面试官区分「会用 ChatGPT」和「能上线 AI 功能」：

| 会聊天 | AI 应用工程 |
|--------|-------------|
| 调一次回答 | 稳定 API、权限、成本、延迟 |
| 凭感觉改提示 | 评测回归、灰度 |
| 不管数据 | RAG/权限/安全 |

## 二、典型架构（P0，要能画）

```text
客户端 → 网关(鉴权/限流) → 应用服务
         → 缓存 → 检索(向量库/ES) → 组装 Prompt → LLM
         → 日志/Trace/评测
```

**面试怎么答：** 分层说清接入、编排、检索、模型、数据、观测；关键路径超时与降级。

## 三、RAG（P0）

### 白话

模型训练后知识会过时，也不会你们的私有文档。  
**RAG：** 先检索相关片段，再让模型「依据材料回答」，降低胡说、可追溯。

流程：解析文档 → 分块 → embedding 入库 → 问题向量化检索 topK →（重排）→ 拼进提示词生成 → 最好带引用。  
分块太大噪声多、太碎没上下文。检索不到应**拒答**，别硬编。

### 面试怎么答

> 问题、流程、分块注意点、引用与拒答、和微调分工（知识变→RAG）。

## 四、幻觉与质量（P0）

手段：引用 grounding、低分拒答、结构化校验、低温、评测集与抽检。  
Agent 还要防「捏造工具结果」——工具必须真执行。

---

## 五、服务化（P0）

模型慢且不稳：超时、有限重试+退避、限流、熔断、流式 SSE、长任务进队列。  
密钥不进仓库；request_id + 分阶段耗时 + token 费用。

**详述：** [AI应用面渣](./AI应用工程面渣级口述.md) · [LLMOps](./LLMOps与可观测高频面试题与知识点.md) · [FastAPI](./Python异步与FastAPI面渣级口述.md)

## 六、何时上 Agent（P1）

步骤固定 → 工作流/RAG 链更稳更便宜。  
要动态选工具、多步反馈 → Agent，并限步、权限、HITL。
