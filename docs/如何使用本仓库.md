# 如何使用本仓库（必读）

很多同学反馈：

> **知识点太简单 / 太干 / 看不懂 / 不知道面试怎么说。**

这不是你「笨」，是两种文档分工不同。按下面用，就不会晕。

---

## 1. 两种文档分别干什么

| 类型 | 文件名 | 作用 | 单独看够不够 |
|------|--------|------|--------------|
| **面渣口述** | `…面渣级口述.md` | **主教材**：完整 2～4 分钟口语稿 | **够** |
| **知识点** | `…知识点.md` | 地图 + 白话 + **可照着说的短答** + 追问 | 可练结构，**仍建议配面渣** |
| **路径表** | `路径-….md` | 先学什么后学什么 | 导航 |

**结论：**

- 想学会 **怎么开口** → **先面渣**（长文通读 + 录音）  
- 想快速过结构、查漏 → 再看知识点  
- 知识点里已尽量写了「面试怎么答（可照着说）」；若仍觉得短，**跳面渣不要硬啃表**

---

## 2. 推荐读法（每天一套）

```text
① 打开路径表，找到今天一步
② 打开「面渣」全文通读（可先听自己读）
③ 对着稿说 2 分钟 → 遮稿再说一遍
④ 打开「知识点」：只看自己卡壳的节
⑤ 做追问三连 / 自测清单
```

**错误用法（最常见）：**

```text
只刷知识点表格关键词 → 面试一开口只能甩名词 → 被追问就沉默
```

**正确心理预期：**

| 你的感觉 | 实际含义 | 动作 |
|----------|----------|------|
| 知识点太简单 | 它本来是提纲 + 短答 | 去读同名 **面渣** |
| 看不懂某张表 | 缺语境 | 面渣对应章节建立感觉 |
| 不知道怎么说 | 还没练口语 | 照「面试怎么答」念，再遮稿 |
| 面渣太长 | 正常 | 按路径 P0 先拿下，别一次通宵 |

---

## 3. 知识点正文怎么读

每个核心考点尽量四段：

1. **白话** — 用人话建立直觉  
2. **面试怎么答** — **可直接照着念** 的 45 秒～2 分钟结构  
3. **追问** — 常被打穿的点  
4. **完整口语稿** — 链到面渣长文  

练习标准：

| 程度 | 标准 |
|------|------|
| 不及格 | 只会甩 HashMap、GC、Agent 等词 |
| 及格 | 能按「面试怎么答」说满 1～2 分钟，偶看稿 |
| 良好 | 不看稿说清 + 接住 2～3 个追问 |
| 优秀 | 能接到自己项目 STAR 举例 |

## 5. 仍觉得某个专题「太薄」

可能原因：

1. 你打开的是知识点不是面渣  
2. 该专题面渣本身偏短（部分 AI 周边专题）— 以知识点「可照着说」+ 追问补练  
3. 需要结合项目 — 看 [STAR 范例](./项目STAR范例-AI.md) / [Java STAR](./项目STAR范例-Java.md)

可以点名专题继续加厚（例如：「只要 Redis 再加长」「Agent 再多两个场景」）。
