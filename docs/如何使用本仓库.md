# 如何使用本仓库（必读）

很多同学反馈：**知识点太干、看不懂、不知道怎么说。**  
这是设计问题，不是你的问题。按下面方式用即可。

---

## 两种文档分别干什么

| 类型 | 文件名里通常有 | 作用 | 单独看够不够 |
|------|----------------|------|--------------|
| **知识点** | `…知识点.md` | 地图、提纲、对照表、考什么 | **不够**，只当目录 |
| **面渣口述** | `…面渣…md` | **完整怎么说**（2～4 分钟口语稿） | **够**，主教材 |
| **路径表** | `路径-….md` | 先学什么后学什么 | 导航用 |

**结论：想学会「怎么答面试」，以面渣口述为主；知识点用来查漏和看结构。**

---

## 推荐读法（三步）

```text
① 打开路径表（AI / Java / Python）找到当前步
② 直接打开「口述」全文通读一遍 → 对着说（可看稿）
③ 再扫一眼「知识点」P0 表，对照自己漏了啥
④ 遮住稿子再说一遍 → 看追问三连
```

**不要：** 只背知识点表格里的关键词，却从没读过口述。  
**那样必然：** 知道名词、不会展开、一追问就卡。

---

## 若仍觉得「知识点太简单」

已在核心专题把知识点改成四段式：

1. **白话** — 用人话讲是什么  
2. **面试怎么答** — 结构提纲（30 秒～1 分钟）  
3. **易混/追问** — 常被问倒的点  
4. **完整口述** — 链到面渣长文  

顺序仍然是：先口述建立感觉，再回知识点巩固结构。

## 练习标准（什么叫会了）

| 程度 | 标准 |
|------|------|
| 不及格 | 只会甩 HashMap、GC 等词 |

| 优秀 | 能结合自己项目举例子 |
