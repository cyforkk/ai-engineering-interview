### 1. Token 和上下文窗口

**口述：**

> 大模型按 token 计费和限长，不是简单按字。中文一个字常对应多个 token，所以同样字数比英文更「吃」窗口。  
> 上下文窗口是一次请求里系统提示、历史、检索片段、用户问题和预留输出的总和上限。  
> 满了要摘要历史、检索只留 top 相关、或截断旧轮次，而不是无限塞。

---

### 2. 温度和幻觉

**口述：**

> 温度高输出更多样，低更确定。客服、制度问答、抽字段宜低温；创意写作可调高。  
> 幻觉是模型生成看似合理但不实的内容，因为它本质是下一个 token 概率，不是查数据库。  
> 工程上靠 RAG 引用、拒答、校验和评测压风险，而不是幻想模型永不胡说。

---

### 3. 预训练、SFT、RLHF

**口述：**

> 预训练在大规模语料上学会语言和知识；SFT 用指令数据教会「按人的要求回答」；RLHF 或类似偏好优化让回答更有用、更安全、更符合人类偏好。  
> 业务里换底座模型、做领域 SFT、上 RAG，是不同成本曲线，面试要会分工。

---

### 4. 开源私有化 vs 云 API

**口述：**

> 云 API 迭代快、运维轻，适合验证产品和一般数据；要看是否出境、是否用于训练。  
> 私有化可控数据与定制，但要 GPU、部署、扩缩容和安全补丁。  
> 很多团队混合：敏感走私有，一般走云，用统一网关抽象。

---

### 5. 成本和延迟指标

**口述：**

> 成本大致是单价 ×（输入 token + 输出 token），缓存和路由能省钱。  
> 延迟看 TTFT 首 token 时间和总耗时；流式改善体感。  
> 线上要分阶段打点：检索、组装、模型、后处理。
