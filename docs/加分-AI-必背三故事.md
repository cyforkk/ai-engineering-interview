## 故事 1：幻觉与「有依据才答」

**场景：** 制度/产品问答，模型编造条款，客服信了出事故。

**你怎么做（工程动作）：**

1. RAG：切块 → 向量+关键词 → 重排 → 生成  
2. **强制引用** chunk id；相似度低于阈值 → **拒答**  
3. Prompt：仅依据材料，无则说不知道  
4. 金标集含「无依据题」，不过门禁不发版  

**指标：** 引用正确率、拒答率、严重幻觉工单数。

**追问：**

- 只有 Prompt 行不行？→ 不行，要检索 grounding + 服务端策略。  
- 材料被投毒？→ 入库审核、权限 filter、间接注入防护。  

**链：** [向量库面渣](./向量库面渣级口述.md) · [评测面渣](./AI评测金标SOP面渣级口述.md)

## 故事 2：工具失败与权限（Agent）

**场景：** Agent 调「查工单/改状态」工具，超时、乱调、编造参数。

**你怎么做：**

1. 工具 **白名单 + JSON Schema**；服务端鉴权（模型不可信）  
2. ReAct **max_steps**、重复动作熔断  
3. 读可重试；写要幂等键；连续失败 → 降级仅文档答或 HITL  
4. 轨迹打点：tool_name、latency、error_code  

**指标：** 工具调用正确率、失败率、平均步数、费用。

**追问：**

- 为何不全自动 Multi-Agent？→ 难测、贵；固定步骤走工作流。  
- 工具返回很长？→ 截断/摘要，防爆上下文。  

**链：** [Agent 面渣](./Agent面渣级口述.md) · [安全面渣](./Prompt注入与AI安全面渣级口述.md)

## 故事 3：金标门禁与发版（LLMOps）

**场景：** 业务要「下周全量新 Prompt」，无评测就上。

**你怎么做：**

1. 金标三类：事实题 / 拒答题 / 工具题  
2. CI：改 Prompt/索引/模型 → 跑离线集 → **不达标禁止发**  
3. 版本：prompt_id + model + index_version 绑定  
4. 灰度 + 在线点踩/费用；可一键回滚上一版本  

**指标：** 离线正确率、线上点踩、P95、$/千次。

**追问：**

- 金标过拟合？→ 收真实差评进集，定期换血。  
- 只看 HTTP 200？→ 胡说也是 200，要质量信号。  

**链：** [评测面渣](./AI评测金标SOP面渣级口述.md) · [LLMOps 面渣](./LLMOps与可观测面渣级口述.md)
