## 1. TCP vs UDP？

- **TCP**：面向连接、可靠、有序；适合 HTTP、DB、RPC。
- **UDP**：无连接、快、不保证可靠；适合 DNS、音视频等。

---

## 2. 三次握手？为什么是三次？

- SYN → SYN+ACK → ACK。
- 同步双方初始序列号；确认双向通信能力。
- 三次有助于处理历史延迟 SYN，降低错误连接风险。

---

## 3. 四次挥手？TIME_WAIT？CLOSE_WAIT？

- 全双工分别关闭 → 常四次挥手。
- **TIME_WAIT**（主动关闭方，约 2MSL）：保证最后 ACK；让旧包过期。
- **CLOSE_WAIT 多**：应用未 close。

---

## 4. HTTPS？

- HTTP + TLS。
- 证书验证身份；非对称换钥 + 对称加密传数据。

---

## 5. 常见状态码？

- **401**：未认证；**403**：无权限。
- **502**：上游异常；**504**：上游超时。

---

## 6. TCP 粘包 / 拆包？

- TCP 是字节流，无消息边界。
- 解决：长度前缀、分隔符、定长。
