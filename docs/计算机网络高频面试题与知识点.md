## 二、TCP vs UDP（P0）

### 白话

- **TCP**：先建连，保证可靠、有序、有流量/拥塞控制。代价是延迟和状态开销。HTTP、MySQL、RPC 大多基于 TCP。  
- **UDP**：不建连、不保证送达与顺序，开销小、延迟低。DNS、很多音视频/实时游戏用 UDP（上层自己做可靠性或容忍丢包）。

**不要说「UDP 没用」**——场景不同。

### 面试怎么答（约 45 秒）

> TCP 面向连接、可靠有序，适合不能丢的业务数据；UDP 无连接、快，适合能容忍丢包或自己做可靠性的场景。  
> Web 业务、数据库、大多数 RPC 用 TCP；DNS 查询、直播推流等常见 UDP。选型看是否要可靠、延迟预算和实现复杂度。

## 三、三次握手（P0）

### 白话

双方都要确认：「我能发、你能收；你能发、我能收」。

```text
客户端 → SYN
服务端 → SYN + ACK
客户端 → ACK
然后进入 ESTABLISHED
```

**为何不是两次：** 若只有两次，旧的延迟 SYN 可能被当成新连接，浪费服务端资源；三次让客户端再确认一次，避免历史连接误建立。也确认了双方初始序列号。

### 面试怎么答（约 1 分钟）

> 三次握手是 TCP 建立连接：客户端发 SYN，服务端回 SYN+ACK，客户端再 ACK。  
> 目的是同步双方初始序列号，并确认双向通信能力。  
> 用三次而不是两次，可以降低历史重复 SYN 导致错误连接的风险，服务端也不会在未确认前就投入过多资源。  
> 面试若追问状态：客户端 SYN_SENT，服务端 LISTEN→SYN_RCVD→ESTABLISHED。

## 四、四次挥手与 TIME_WAIT（P0）

### 白话

TCP 全双工：每个方向都要单独关闭。

```text
主动方 FIN → 被动方 ACK（被动方可能还有数据要发）
被动方发完再 FIN → 主动方 ACK
```

所以常是 **四次**（中间两次 ACK/FIN 有时可合并成三次，但原理按四次讲更清楚）。

**TIME_WAIT：** 主动关闭方最后等 2MSL。  
作用：1）保证最后的 ACK 若丢了还能重传场景下对端能正确结束；2）让旧连接的包在网络里「过期」，不污染新连接。

**CLOSE_WAIT 很多：** 通常是 **应用没调用 close**，被动方已收到 FIN 但进程没关连接。

### 面试怎么答

> 四次挥手因为全双工要分别关闭两个方向。  
> TIME_WAIT 在主动关闭方，持续约 2MSL，保护最后 ACK 并清空旧报文。  
> 若服务器出现大量 CLOSE_WAIT，优先查应用是否泄漏连接、没正确关闭。  
> 大量 TIME_WAIT 在高并发短连接场景常见，可从连接复用、参数与架构上优化，不要盲改内核参数。

## 五、HTTPS（P0）

### 白话

HTTP 明文；HTTPS ≈ HTTP over TLS。  
证书（PKI）证明「你连的是真的目标站」；握手阶段用非对称密码学换出对称密钥，之后大数据用对称加密传（更快）。

### 面试怎么答

> HTTPS 在 HTTP 下加 TLS：证书校验服务器身份，握手协商出对称密钥，后续加密通信，防窃听和篡改。  
> 还能说明公钥加密换钥、对称加密传数据，避免全程非对称的性能问题。

## 六、状态码（P0 速答）

| 码 | 怎么说 |
|----|--------|
| 200 | 成功 |
| 301/302 | 永久/临时重定向 |
| 400 | 请求本身有问题 |
| **401** | **没认证**（未登录/Token 无效） |
| **403** | **认证了但没权限** |
| 404 | 资源不存在 |
| 500 | 服务端未处理异常 |
| **502** | 网关/反向代理上游异常 |
| **504** | 网关等上游超时 |

### 面试怎么答

> 401 和 403 最容易混：401 是身份没过，403 是身份有了但授权不够。  
> 502/504 常出在 Nginx 反代：502 上游挂了或协议错，504 是上游太慢超时。

## 七、TCP 粘包 / 拆包（P1）

### 白话

TCP 是 **字节流**，没有「消息边界」。你 write 两次，对端可能一次 read 到两段拼在一起（粘包），或一次只读到半段（拆包）。

**解法：** 应用层协议定边界——长度前缀、分隔符、定长消息。

### 面试怎么答

> TCP 保证字节可靠有序，但不保留消息边界，所以有粘包拆包。  
> 协议层用长度头或分隔符解决；HTTP 有 Content-Length/chunked，自己写 RPC 也要定帧。
