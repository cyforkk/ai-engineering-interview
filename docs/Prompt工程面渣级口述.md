# Prompt 工程 · 面渣级口述（加长）

> [知识点](./Prompt工程高频面试题与知识点.md) · [AI完整卷](./AI-Agent高频面试题与知识点.md)

---

## 一、定义

Prompt 工程是把业务规则写成模型可执行的输入契约：角色、边界、步骤、输出格式、示例，并 **模板化、版本化、可评测**。

---

## 二、生产级结构（2 分钟）

1. **系统角色与目标**  
2. **硬边界**：不能编造；材料不足输出「不知道」或拒答码  
3. **输入槽位**：用户问题、检索文档、工具结果分开，贴标签  
4. **输出合同**：JSON 字段、枚举值、错误码  
5. **少样本**：1～3 个示范，稳格式  
6. **安全**：用户内容当数据，不当系统指令（防注入）  

变量化之后才能回归测试：改一句 Prompt 跑金标，看指标。

---

## 三、和 RAG、Agent

- RAG：明确「仅依据下列资料，引用编号」  
- Agent：规定工具使用规则、最大步数、失败怎么办  
- 温度：事实任务偏低  

---

## 四、坏例子与好例子

坏：规则矛盾、无格式、无拒答出口、系统与用户糊成一团。  
好：像 API 文档，输入输出清晰，可被程序解析校验。

---

## 五、收尾

Prompt 是配置不是玄学；要进 Git、进评测、能回滚。结合一次「改 Prompt 前后指标变化」最加分。
