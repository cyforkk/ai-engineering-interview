# 一、高频核心

## 1. 核心作用？

| 场景 | 说明 |
|------|------|
| 配置中心 | 配置变更通知 |
| 注册中心 | 服务上下线（历史 Dubbo 等） |
| 分布式锁 | 临时顺序节点 |
| 选主 | 主从选举 |
| 集群协调 | Kafka 旧版依赖（KRaft 后弱化） |

## 2. ZAB 协议？

- Zookeeper Atomic Broadcast：崩溃恢复 + 消息广播  
- 与 Paxos/Raft 同属共识族；面试说：**保证副本一致的原子广播**，类似 Raft 选主+日志复制直觉  

## 3. 节点类型？

| 类型 | 特点 |
|------|------|
| 持久 Persistent | 客户端断也不删 |
| 临时 Ephemeral | 会话结束删除 |
| 持久/临时 + 顺序 Sequential | 节点名带序号 |

## 4. 分布式锁？问题？

```text
创建临时顺序节点 → 序号最小获得锁
 → Watch 前一个节点
 → 释放即删节点
```

- **羊群效应**：所有人 watch 同一节点，释放时惊群 → 改为只 watch 前驱  
- 性能与运维不如专用组件时可用 Redis/etcd  

## 5. Watch 机制？

- 客户端注册监听；节点变化 **一次性通知**（需重新注册）  
- 用于配置变更、锁前驱变化

# 二、中频

## 6. 集群台数？

- 至少 3；**奇数**利于多数派（2f+1 容 f 故障）  
- 2 台无意义（挂 1 台无法多数）  

## 7. 会话与重连

- Session；超时则临时节点删  
- 客户端自动重连，需处理会话过期
