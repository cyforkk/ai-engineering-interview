# 一、高频核心

## 1. 为什么性能高？

1. **事件驱动** + 非阻塞 IO  
2. **epoll**（Linux）多路复用  
3. **多进程**：master-worker；worker 常与 CPU 核相当  
4. 连接处理高效、内存池  

## 2. 正向代理 vs 反向代理？

| | 正向 | 反向 |
|--|------|------|
| 代理谁 | 客户端 | 服务端 |
| 例 | 科学上网、公司出口代理 | Nginx 反代 Tomcat/API |

## 3. 负载均衡策略？

| 策略 | 说明 |
|------|------|
| 轮询 | 默认 |
| 加权轮询 | weight |
| ip_hash | 同 IP 会话粘滞 |
| least_conn | 最少连接 |
| 一致性哈希等 | 插件/扩展 |

## 4. location 匹配优先级（简化）

```text
精确 = 
前缀 ^~ （命中则不再正则）
正则 ~ / ~* （按配置顺序）
普通前缀 /
```

- 面试说清：**精确 > ^~ > 正则顺序 > 最长前缀** 的常见口诀。  

## 5. 限流？

| 指令 | 作用 |
|------|------|
| limit_req | 请求速率（漏桶/令牌类配置） |
| limit_conn | 并发连接数 |

## 6. Nginx vs LVS vs HAProxy？

| | Nginx | LVS | HAProxy |
|--|-------|-----|---------|
| 层 | 7 层强（也支持 4） | 4 层 | 4/7 |
| 场景 | 反代、静态、SSL | 超高并发 4 层 | 负载均衡成熟 |
| 性能 | 高 | 更高（内核） | 高 |

# 二、中频

- **动静分离**：静态 Nginx，动态反代应用  
- **缓存** proxy_cache  
- **HTTPS**：证书、TLS 终止  
- master 管配置/worker；worker 处理连接
