## 1. HashMap（JDK8）

```mermaid
flowchart TB
  subgraph table["Node[] table 容量 2^n"]
    B0["桶0"]
    B1["桶1 链: A→B→C"]
    B2["桶2 树化红黑树"]
    Bn["..."]
  end
  Key["key"] --> Hash["hash 扰动"]
  Hash --> Idx["下标 = (n-1) & hash"]
  Idx --> table
```

**口述锚点：**

```text
put: hash → 下标 → 空放/equals覆盖/链尾插
链长≥8 且 表长≥64 → 红黑树
size > 容量×0.75 → 扩容 2 倍
多线程不安全 → ConcurrentHashMap
```

**白板：** 画一排格子 + 一条链 + 一个树图标。  
**完整卷：** [集合](./Java集合框架高频面试题与知识点.md)

## 2. TCP 三次握手 / 四次挥手

### 三次握手

```mermaid
sequenceDiagram
  participant C as 客户端
  participant S as 服务端
  C->>S: SYN seq=x
  S->>C: SYN+ACK seq=y ack=x+1
  C->>S: ACK ack=y+1
  Note over C,S: ESTABLISHED
```

**为何三次：** 同步双方序号 + 确认收发能力；防历史 SYN。

### 四次挥手 + TIME_WAIT

```mermaid
sequenceDiagram
  participant A as 主动关闭
  participant B as 被动关闭
  A->>B: FIN
  B->>A: ACK
  Note over B: 可能还有数据
  B->>A: FIN
  A->>B: ACK
  Note over A: TIME_WAIT 2MSL
```

**TIME_WAIT：** 保最后 ACK；清旧包。  
**CLOSE_WAIT 多：** 应用未 close。  
**完整卷：** [网络](./计算机网络高频面试题与知识点.md)

## 3. Spring 三级缓存（循环依赖）

```mermaid
flowchart LR
  subgraph L3["三级 singletonFactories"]
    F["ObjectFactory\n可生成早期引用/代理"]
  end
  subgraph L2["二级 earlySingletonObjects"]
    E["半成品实例"]
  end
  subgraph L1["一级 singletonObjects"]
    C["成品 Bean"]
  end
  F -->|被循环引用时 getObject| E
  E --> C
```

**A→B→A 口述：**

```text
创建A → 工厂进三级
A 要 B → 创建B
B 要 A → 从三级拿A早期引用
B 完成进一级 → A 填完进一级
```

**解不了：** 构造器循环、prototype。  
**完整卷：** [Spring](./Spring高频面试题与知识点.md)

## 5. RAG Pipeline

```mermaid
flowchart LR
  D[文档] --> C[切片 Chunk]
  C --> E[Embedding]
  E --> V[(向量库)]
  Q[问题] --> EQ[Query Embed]
  EQ --> R[检索 topK]
  V --> R
  R --> RR[Rerank 可选]
  RR --> P[拼 Prompt]
  P --> L[LLM 生成]
  L --> Ans[带引用答案]
```

**防幻觉：** 引用、拒答、阈值、评测。  
**Hybrid：** 向量 + 关键词。  
**完整卷：** [AI-Agent·RAG](./AI-Agent高频面试题与知识点.md) · [向量库](./向量库高频面试题与知识点.md)
