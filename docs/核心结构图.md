## 1. HashMap（JDK8）

```mermaid
flowchart TB
  subgraph table["Node[] table 容量 2^n"]
    B0["桶0"]
    B1["桶1 链: A→B→C"]
    B2["桶2 树化红黑树"]
    Bn["..."]
  end
  Key["key"] --> Hash["hash 扰动"]
  Hash --> Idx["下标 = (n-1) & hash"]
  Idx --> table
```

**口述锚点：**

```text
put: hash → 下标 → 空放/equals覆盖/链尾插
链长≥8 且 表长≥64 → 红黑树
size > 容量×0.75 → 扩容 2 倍
多线程不安全 → ConcurrentHashMap
```

**白板：** 画一排格子 + 一条链 + 一个树图标。  
**完整卷：** [集合](./Java集合框架高频面试题与知识点.md)

## 2. TCP 三次握手 / 四次挥手

### 三次握手

```mermaid
sequenceDiagram
  participant C as 客户端
  participant S as 服务端
  C->>S: SYN seq=x
  S->>C: SYN+ACK seq=y ack=x+1
  C->>S: ACK ack=y+1
  Note over C,S: ESTABLISHED
```

**为何三次：** 同步双方序号 + 确认收发能力；防历史 SYN。

### 四次挥手 + TIME_WAIT

```mermaid
sequenceDiagram
  participant A as 主动关闭
  participant B as 被动关闭
  A->>B: FIN
  B->>A: ACK
  Note over B: 可能还有数据
  B->>A: FIN
  A->>B: ACK
  Note over A: TIME_WAIT 2MSL
```

**TIME_WAIT：** 保最后 ACK；清旧包。  
**CLOSE_WAIT 多：** 应用未 close。  
**完整卷：** [网络](./计算机网络高频面试题与知识点.md)

## 3. Spring 三级缓存（循环依赖）

```mermaid
flowchart LR
  subgraph L3["三级 singletonFactories"]
    F["ObjectFactory\n可生成早期引用/代理"]
  end
  subgraph L2["二级 earlySingletonObjects"]
    E["半成品实例"]
  end
  subgraph L1["一级 singletonObjects"]
    C["成品 Bean"]
  end
  F -->|被循环引用时 getObject| E
  E --> C
```

**A→B→A 口述：**

```text
创建A → 工厂进三级
A 要 B → 创建B
B 要 A → 从三级拿A早期引用
B 完成进一级 → A 填完进一级
```

**解不了：** 构造器循环、prototype。  
**完整卷：** [Spring](./Spring高频面试题与知识点.md)

## 6. 线程池提交流程

```mermaid
flowchart TD
  Task[提交任务] --> C{线程数 < core?}
  C -->|是| Core[创建核心线程执行]
  C -->|否| Q{队列是否已满?}
  Q -->|否| Offer[入队等待]
  Q -->|是| M{线程数 < max?}
  M -->|是| Extra[创建非核心线程]
  M -->|否| R[拒绝策略]
```

**口述锚点：**

```text
core → 队列 → max → 拒绝
拒绝：Abort / CallerRuns / Discard / DiscardOldest
不要用 Executors 默认无界队列（易 OOM）
```

**完整卷：** [并发](./并发高频面试题与知识点.md)

## 7. synchronized 锁升级

```mermaid
flowchart LR
  N[无锁] --> B[偏向锁]
  B --> L[轻量级锁\n自旋/CAS]
  L --> H[重量级锁\nMonitor 阻塞]
```

**口述锚点：**

```text
无竞争：偏向，减 CAS
轻竞争：自旋
重竞争：阻塞进操作系统互斥
对象头 Mark Word 记锁状态
```

## 8. MySQL B+ 与回表

```mermaid
flowchart TB
  subgraph sec["二级索引 B+"]
    S1[叶子: 二级键 + 主键]
  end
  subgraph clu["主键聚簇索引 B+"]
    C1[叶子: 主键 + 整行]
  end
  Q[WHERE 二级列] --> sec
  sec -->|回表| clu
  Cov[覆盖索引: 查询列都在二级] --> NoBack[无需回表]
```

**口述锚点：**

```text
聚簇叶子=整行；二级叶子=主键 → 常回表
覆盖索引免回表
联合索引最左前缀 (a,b,c)
```

**完整卷：** [MySQL](./MySQL高频面试题与知识点.md)

## 9. 秒杀漏斗

```mermaid
flowchart TD
  U[海量请求] --> FE[前端防抖/验证/静态化]
  FE --> GW[网关限流]
  GW --> SVC[服务校验/限购]
  SVC --> Redis[Redis Lua 原子扣库存]
  Redis --> MQ[MQ 异步下单]
  MQ --> DB[DB 条件更新兜底]
  DB --> OK[订单成功]
  Redis -->|库存不足| Fail[失败返回]
```

**口述锚点：**

```text
层层过滤 → Redis 预扣扛并发 → MQ 削峰 → DB 防超卖
幂等单号防重复下单
对账回补库存
```

**完整卷：** [系统设计](./后端系统设计高频面试题与知识点.md) · [场景题](./场景题高频面试题与知识点.md)
