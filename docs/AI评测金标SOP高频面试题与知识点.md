## 二、金标集怎么建（SOP）

### 步骤

```text
1. 定义任务与成功标准（准确/拒答/格式/安全）
2. 从真实日志抽样（去敏）+ 人工构造边界例
3. 标注：标准答案 / 可接受答案要点 / 必引用来源 / 应拒答
4. 分层：简单 / 普通 / 困难 / 对抗（注入）
5. 冻结版本 golden_v1；变更走评审
6. 接入 CI：分数低于门槛 fail
7. 线上坏例回流进集合
```

### 规模建议

| 阶段 | 条数（参考） |
|------|----------------|
| 起步 | 50～100 |
| 可用 | 200～500 |
| 稳健 | 1000+ 且持续增 |

质量重于数量：每条可判定、可复现。

### 标注字段示例

```text
id, query, context_ids?, expected_answer_points[],
must_cite[], should_refuse, tags[], severity
```

## 三、指标（按任务选）

| 层 | 指标 |
|----|------|
| 检索 | Recall@K、MRR、权限误召回=0 |
| 生成 | 忠实度、要点覆盖、格式合法率 |
| 安全 | 注入攻击成功率（越低越好） |
| 业务 | 人工满意度、工单解决率 |
| 工程 | 延迟、费用（作辅助门槛） |

**自动化：** 规则校验 JSON；关键词/要点匹配；LLM-as-Judge（要校准）；关键集人工。

---

## 四、发版门槛（示例）

```text
- 忠实度 ≥ 基线 - 1%
- 格式合法率 = 100%（结构化任务）
- 安全集攻击成功 = 0（或 ≤ 基线）
- P99 延迟 ≤ SLA
- 单次成本 ≤ 预算
```

不通过：**禁止全量**；只准灰度实验环境。

## 五、回归流程

```text
改 Prompt/模型/分块/检索
  → 跑 golden_vN
  → 出 diff 报告（哪几题挂了）
  → 人工看失败 case
  → 修或下调预期并改标注
  → 合并
```
