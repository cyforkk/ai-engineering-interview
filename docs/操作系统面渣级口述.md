## P0 长口述

### 1. 进程和线程区别

**口述：**

> 进程是资源分配的基本单位，有独立地址空间，进程间默认隔离。  
> 线程是 CPU 调度的基本单位，同一进程内线程共享堆、全局变量等，各自有栈。  
> 创建切换线程比进程便宜，因为不用换页表等那么重。  
> 线程共享内存通信方便，但要同步，一个线程踩内存可能搞崩整个进程。  
> 进程隔离更好，崩溃相对不影响别的进程。  
> 所以：要隔离和稳定性用多进程；要轻量并发共享数据用多线程。Java 里我们直接面对的是线程，进程是 JVM 进程。

---

### 2. 用户态和内核态

**口述：**

> CPU 有特权级，用户程序跑在用户态，不能随便碰硬件和关键内核数据。  
> 要执行受限操作就通过**系统调用**陷入内核态，由内核完成再返回。  
> 文件 IO、网络、创建线程都会涉及系统调用。  
> 频繁在用户态内核态切换有成本，所以会有缓冲、批量、零拷贝等优化减少切换和拷贝。

---

### 3. 上下文切换

**口述：**

> 切换线程或进程时，要保存当前寄存器、程序计数器等，加载下一个的，叫上下文切换。  
> 切换太频繁有效计算时间变少，CPU 高但业务吞吐上不去。  
> 线程池复用线程就是为了减少创建销毁和部分切换开销；也要避免线程数远超 CPU 导致狂切。

## P1 长口述

### 4. select / poll / epoll

**口述：**

> 都是 IO 多路复用：一个线程监听多个 fd。  
> select 有 fd 数量限制，每次要把 fd 集合从用户态拷到内核，内核线性扫描。  
> poll 去掉数量限制，但仍是线性扫描，连接多了慢。  
> epoll 用事件就绪队列，适合海量连接，Linux 高性能网络标配。  
> Nginx、Redis、Netty 底层思想都和「多路复用 + 事件驱动」相关。  
> 和阻塞 BIO「一连接一线程」比，能用更少线程撑更多连接。

---

### 5. 死锁

**口述：**

> 多个执行流互相持有对方需要的资源，都无法推进。  
> 四条件：互斥、占有且等待、不可抢占、循环等待。  
> 破坏任一条件即可：统一加锁顺序破循环等待；trylock 超时；减少持锁时间。  
> Java 里 jstack 可查死锁，原理和 OS 死锁一样。

---

### 6. 虚拟内存

**口述：**

> 进程以为自己有连续完整地址空间，实际通过页表映射到物理页，这叫虚拟内存。  
> 好处：隔离保护、可用内存可以大于物理（换页）、方便加载。  
> 访问不在内存的页会缺页中断，内核调页，可能很慢。  
> JVM 堆、栈都在这个虚拟地址空间里，GC 管的是堆里对象，物理页面仍由 OS 管。

---

### 7. 同步异步、阻塞非阻塞

**口述：**

> 阻塞：调用直到完成才返回。  
> 非阻塞：调用立即返回，可能告诉你还没好。  
> 同步异步更强调完成通知方式：异步是完成后通知或回调。  
> 可以组合，比如同步非阻塞。面试别和 Java NIO 名词死磕定义，结合例子说清楚。
