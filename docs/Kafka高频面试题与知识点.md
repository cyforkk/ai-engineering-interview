## 二、核心模型 P0

| 概念 | 说明 |
|------|------|
| Topic | 逻辑主题 |
| Partition | 并行单位；有序在**分区内** |
| Producer | 按 key 路由分区 |
| Consumer Group | 组内分摊分区；组间广播 |
| Offset | 消费进度 |
| Broker / ISR | 副本同步集合 |

### 为什么快（口述点）

- 顺序写磁盘、零拷贝、批量、分区并行  
- 页缓存  

### 可靠性

| 点 | 说明 |
|----|------|
| acks=all | 等 ISR 确认，更安全更慢 |
| 副本 | replication.factor ≥ 2/3 |
| 生产者重试 | 可能重复 → 消费幂等 |
| 消费者 | 处理成功再提交 offset；注意 auto commit |

### 顺序

- 同 key 同分区 → 分区内有序  
- 多分区全局无序  
- 单分区吞吐受限  

---

## 三、P1 常考

| 点 | 说明 |
|----|------|
| Rebalance | 成员变导致分区重分配；停顿与重复消费风险 |
| 积压 | lag 监控；扩分区/消费者；优化消费 |
| 幂等生产者 | 防网络重试重复写（broker 内） |
| 事务 | 读已提交等，跨分区原子，成本高 |
| 语义 | at-least-once 常见；exactly-once 有条件 |

### 延时消息

Kafka 无任意延时：时间轮/外部调度/分层 topic（见延时消息口述）。
