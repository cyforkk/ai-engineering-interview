## 和通用 MQ 题的关系

| 通用考点（完整卷） | Kafka 上怎么落地 |
|--------------------|------------------|
| 不丢 | `acks=all` + 足够 ISR；生产重试；消费处理成功再提交 offset |
| 不重 | 至少一次语义 → 业务幂等；幂等生产者 / 事务（成本高） |
| 顺序 | **同 key → 同分区** + 分区内单线程消费 |
| 积压 | 先加分区再加消费者；优化消费；临时扩容 |
| 削峰解耦 | Topic 承载峰值；Consumer Group 水平扩展 |

---

## Kafka  diff（只记这些）

| 点 | 一句话 |
|----|--------|
| 分区 | 并行与有序的基本单位 |
| Consumer Group | 组内分摊分区；组间广播式独立进度 |
| offset | 消费进度；自动提交易丢/重 |
| ISR / HW | 同步副本集合；高水位决定可见性 |
| rebalance | 成员变更时重新分配分区，期间可能停消费 |
| 副本 | leader/follower；`min.insync.replicas` 与 acks 配合 |
| KRaft | 新版本去 ZooKeeper（了解） |

## 选型一句

- **日志/大数据吞吐、可重放、分区扩展** → 偏 Kafka  
- 业务事务消息/延时/易运维中文文档 → 看 RocketMQ（完整卷对比表）
