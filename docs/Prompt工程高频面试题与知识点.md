## 二、P0 结构模板

```text
1. 角色：你是谁
2. 任务：要完成什么
3. 上下文：已知信息（检索/用户资料）
4. 约束：不能做什么、语气、长度
5. 输出格式：JSON/Markdown/字段列表
6. 示例（可选 few-shot）
7. 用户问题
```

| 原则 | 说明 |
|------|------|
| 具体 | 少空泛「专业一点」 |
| 可验证 | 输出可校验字段 |
| 分步 | 复杂任务拆步骤或先计划后答 |
| 边界 | 无信息时拒答/追问 |

### 常用技巧

| 技巧 | 场景 | 注意 |
|------|------|------|
| Few-shot | 格式、分类、风格 | 示例要覆盖边界 |
| CoT / 逐步思考 | 数学、逻辑 | 延迟与泄露推理；可用隐藏思考 |
| 角色扮演 | 客服/专家 | 防越权与注入（安全篇） |
| 输出 Schema | 抽字段、API | 校验失败重试 |
| 分解 | 长文档 | 多步流水线优于一锅炖 |

### RAG 拼接

```text
系统：规则 + 拒答策略
用户：问题
上下文：【文档1】…【文档2】…（标明仅作参考）
要求：只依据上下文；引用编号；无则说不知道
```

## 三、P1 工程化

| 点 | 实践 |
|----|------|
| 模板 | Jinja/语言 f-string；变量转义 |
| 版本 | prompt_id + version；与评测绑定 |
| 配置中心 | 热更新要灰度 |
| A/B | 同一金标集对比 |
| 多语言 | 系统语言与用户语言策略 |
| 长度 | 动态裁剪上下文，保留最近+最相关 |
