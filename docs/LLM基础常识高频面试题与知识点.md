## 一、要搞定什么

AI 岗开场常问「你怎么理解大模型」——不是要你推公式，而是：

- 计费与长度用 **token**，不是字数  
- **会胡说（幻觉）**，工程要 grounding  
- 温度等参数调「稳还是活」  
- 预训练 / SFT / 偏好 各干什么

## 二、Token 与上下文（P0）

### 白话

模型把文本切成 **token**（子词片）。中文往往「字更吃 token」。  
API 按输入+输出 token 计费；上下文窗口是一次能塞下的上限（系统提示+历史+检索+问题+预留输出）。  
满了要：摘要历史、少塞检索、截断旧轮，而不是无限拼。

### 面试怎么答

> token 是计费与长度单位；窗口有上限；中文更耗；超窗要压缩策略。

## 三、温度与幻觉（P0）

### 白话

**温度高：** 更随机、更「有创意」。  
**温度低：** 更确定，适合抽字段、答制度、客服。  

**幻觉：** 说得流畅但不真实。模型是在预测下一个 token，不是查数据库。  
所以要 RAG 引用、低相关拒答、校验、评测——不能幻想「换个模型永不胡说」。

### 面试怎么答

> 温度调随机性；事实任务偏低。  
> 幻觉是生成特性；工程用 grounding+拒答+评测压风险。

## 四、预训练 / SFT / RLHF（P0）

| 阶段 | 白话 |
|------|------|
| 预训练 | 海量文本学会语言和知识 |
| SFT | 指令数据教会「按人的要求答」 |
| RLHF/偏好 | 更有用、更符合人类偏好与安全 |

业务：知识常变 → RAG；行为风格 → 可微调或强 Prompt。

## 五、云 API vs 私有化

云：快、运维轻，看数据是否出域。  
私有：可控，要 GPU 与运维。  
常混合：网关统一，敏感走私有。
