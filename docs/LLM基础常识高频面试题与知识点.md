## 二、P0 必会

### 2.1 Token 与上下文

| 概念 | 工程含义 |
|------|----------|
| Token | 计费与长度单位，≠ 汉字个数 |
| Context window | 单次能塞进的大致上限（系统+历史+检索+输出） |
| 超窗 | 截断、摘要、滑窗、RAG 减负 |
| 输出 max tokens | 控制费用与截断 |

### 2.2 采样参数

| 参数 | 作用 |
|------|------|
| temperature | 高更随机，低更稳；事实问答宜偏低 |
| top_p / top_k | 核采样/截断候选 |
| seed | 部分 API 可复现（非绝对） |

### 2.3 能力与边界

| 点 | 结论 |
|----|------|
| 幻觉 | 流畅≠正确；要 grounding |
| 知识截止 | 训练后新事实不知；靠 RAG/工具 |
| 推理 | 强模型更好，但仍会错；关键链路校验 |
| 工具 | 实时与行动靠 Function Calling / Agent |

### 2.4 训练阶段（口述级）

| 阶段 | 一句话 |
|------|--------|
| 预训练 | 海量文本学语言与世界知识 |
| SFT | 指令微调，学会「按要求答」 |
| RLHF/偏好 | 更符合人类偏好与安全 |
| 继续预训练/领域 | 领域语料适配（成本高） |

### 2.5 调用形态

| 形态 | 场景 |
|------|------|
| 云 API | 快上线，数据出域 |
| 私有化 | 数据敏感，运维重 |
| 批推理 vs 在线 | 离线便宜；在线要延迟 SLA |

## 三、P1

| 点 | 说明 |
|----|------|
| Embedding | 文本→向量，用于检索 |
| Tokenizer | 文本切成 token 的规则/词表 |
| 多模态 | 图文输入输出；解析链路更复杂 |
| 结构化输出 | JSON mode / schema；失败重试 |
| 流式 | 首 token 延迟 TTFT 体验 |
