# 实现记录：知识点「太简单 / 不会说」问题

## 问题

用户反馈：知识点太简单、太简洁，看完仍不知道面试怎么说。

## 根因

1. **知识点**定位是提纲，不是完整口语教材  
2. 部分专题（OS、网络、Agent 等）知识点过短，只有骨架词  
3. 若用户只刷知识点、不读 **面渣口述**，必然「会词不会说」

## 处理

1. 强化 [如何使用本仓库.md](./如何使用本仓库.md)：错误用法 / 正确读法 / 感觉对照表  
2. 将偏薄核心知识点加厚为四段式：**白话 → 面试怎么答（可照着说）→ 追问 → 链面渣**  
3. 涉及：`操作系统`、`计算机网络`、`Agent`、`Redis`、`MySQL`、`LLM基础`、`Python`  
4. 首页再次标明：觉得太简单 → 跳面渣

## 使用方式

- 主练嘴：路径表左侧 **面渣**  
- 理结构：知识点中的「面试怎么答」可当 1～2 分钟稿  
- 仍薄的专题：点名继续加长面渣或知识点
