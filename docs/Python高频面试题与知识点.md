## 一、对象模型：可变与默认参数

一切皆对象。可变对象可原地改；不可变「改」往往产生新对象。  

**默认参数在定义时绑定**：`def f(a=[])` 的 `[]` 只建一次，多次调用共享 → 串数据。  
所以然：这是语言语义，不是偶然 bug。用 `None` 作哨兵，函数体内新建。

浅拷贝只复制容器外壳；嵌套可变对象仍共享 → 需深拷贝才独立。

## 二、装饰器 / 生成器 / with

| 特性 | 解决什么问题 |
|------|----------------|
| 装饰器 | 无侵入增加日志、鉴权、重试；`wraps` 保留元数据便于调试 |
| 生成器 | 惰性产出，大文件/流式数据不一次进内存 |
| with | 异常路径也执行清理，消灭漏关文件/锁 |

## 三、GIL 与并发选型（核心）

CPython GIL：同进程同时多一线程执行 Python 字节码，简化内存管理。

| 负载 | 策略 | 所以然 |
|------|------|--------|
| CPU 密集 | 多进程 / 外部计算服务 | 多线程吃不满多核 |
| IO 密集 | 线程或 asyncio | 等待时释放 GIL 或让出循环 |
| 高并发 HTTP/LLM | asyncio + 异步客户端 | 单线程交错大量等待 |

**asyncio 禁忌：** `time.sleep`、同步 `requests`、重 CPU 循环会**堵死事件循环**，所有协程一起卡。应用 `asyncio.sleep`、httpx async、进程池卸载计算。

FastAPI：类型注解 + async + OpenAPI，适合 AI 网关；仍需自己做超时、限流、连接池与观测。详见 [Python异步与FastAPI面渣级口述.md](./Python异步与FastAPI面渣级口述.md)。
