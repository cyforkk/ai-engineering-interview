## 二、P0 概念

| 概念 | 说明 |
|------|------|
| 在线推理 | 低延迟 API，流式 |
| 离线批推理 | 吞吐优先，便宜 |
| 显存 | 模型权重 + KV cache + 激活，限制并发 |
| 量化 | INT8/INT4 等减显存提速，略损精度 |
| Batching | 拼 batch 提 GPU 利用率，可能增延迟 |
| 连续批处理 | 动态组 batch（引擎能力） |

### 部署形态

```text
客户端 → 业务 API → 推理网关 → 推理引擎(GPU) → 模型权重存储
```

业务与推理分离：扩缩容、故障隔离、多模型路由。

## 三、P1 引擎与选型（名词级）

| 名称 | 一句话 |
|------|--------|
| vLLM | 高吞吐开源推理，PagedAttention 等优化 |
| TGI | HF 文本生成推理 |
| TensorRT-LLM | 英伟达向极致性能 |
| Ollama | 本地轻量玩模型，非大规模生产首选 |
| 云托管 | 省运维，贵/受限 |

**面试：** 简历写哪个深哪个；没写就说清「业务网关 + 推理服务」分层。

### 容量粗算思路

- 单卡能跑的并发 ≈ 显存与 KV 预算  
- 目标 QPS × 平均延迟 → 需要副本数  
- 留峰值余量；冷启动与滚动更新策略
