## 二、索引（P0）

### 白话

没有索引：像一本书没有目录，找内容只能整本翻（全表扫）。  
有索引：B+ 树像多层目录，很快定位到行。

**InnoDB：** 主键是**聚簇索引**，叶子就是整行数据。  
二级索引叶子存的是**主键值**，所以常要**回表**再查一次聚簇；若查询列都在二级索引里叫**覆盖索引**，少一次回表。

**联合索引最左前缀：** (a,b,c) 要从 a 开始用；只 b 通常用不上这个索引。

**失效常见：** 对索引列套函数、类型隐式转换、`LIKE '%x'`、违反最左、优化器觉得全表更便宜。最终以 **explain** 为准。

### 面试怎么答

> InnoDB B+ 树；聚簇/二级/回表/覆盖；最左前缀；失效情况 + explain。

**完整稿：** [MySQL面渣 · 索引](./MySQL面渣级口述.md)

## 三、事务与隔离、MVCC（P0）

### 白话

事务：一组操作要么都成功要么都回滚（原子），提交后持久，中间要隔离。

**脏读：** 读到别人没提交的。  
**不可重复读：** 同一事务两次读同一行，中间被改了。  
**幻读：** 范围读的行数变了（插入/删除）。

MySQL InnoDB **默认 RR（可重复读）**。

**MVCC：** 多版本。行有版本链（undo），普通 SELECT 按 ReadView 看自己该看哪个版本，叫快照读，一般不用加锁。  
`SELECT ... FOR UPDATE` 是当前读，要加锁。

### 面试怎么答

> ACID；四隔离级别与三种读问题；默认 RR。  
> MVCC：版本链+ReadView；快照读 vs 当前读。

**完整稿：** [MySQL面渣 · 事务/MVCC](./MySQL面渣级口述.md)

## 四、锁与日志（P1）

**行锁**加在索引上，没索引可能锁更狠。RR 下间隙锁防幻读。死锁会回滚一方。

**redo** 持久崩溃恢复；**undo** 回滚与 MVCC；**binlog** 复制与归档；二者一致靠两阶段提交思想。

**完整稿：** [MySQL面渣](./MySQL面渣级口述.md)
