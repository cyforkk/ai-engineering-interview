## 二、P0 必会

### 2.1 索引（第一优先级）

| 点 | 结论 |
|----|------|
| 结构 | InnoDB **B+ 树**（范围友好、矮胖少 IO） |
| 聚簇 | 主键叶子=整行 |
| 二级 | 叶子=主键 → 可能**回表** |
| 覆盖 | 查询列都在索引中，不回表 |
| 最左 | `(a,b,c)` 用 a / ab / abc |
| 失效 | 函数、隐式转换、`%xx`、违最左、选择性差等 |

**自增主键：** 顺序写、页分裂少；二级索引叶子更小。

**高频题**

1. 为什么用 B+ 不用 Hash/B 树？  
2. 回表、覆盖索引？  
3. 最左前缀？  
4. 索引何时失效？  
5. 索引越多越好吗？  

**口述：**  
> B+ 叶子有序链表适合范围；联合索引守最左；能覆盖就少回表；失效先 explain。

---

### 2.2 事务与隔离（P0）

| 级别 | 脏读 | 不可重复读 | 幻读 |
|------|------|------------|------|
| RU | Y | Y | Y |
| RC | N | Y | Y |
| **RR（默认）** | N | N | 控制较好 |
| 串行化 | N | N | N |

- 脏读：读未提交  
- 不可重复读：同行两次结果变（更新）  
- 幻读：范围行数变（插入）  

**高频题：** 默认隔离级别？三种读问题？ACID？

---

### 2.3 MVCC（P0）

| 点 | 结论 |
|----|------|
| 手段 | 版本链（undo）+ ReadView |
| 快照读 | 普通 SELECT |
| 当前读 | FOR UPDATE / 更新删除 |
| RC | 每句新 ReadView |
| RR | 事务内 ReadView 相对稳定 |

**高频题：** MVCC 原理？快照读 vs 当前读？RR 如何尽量避免幻读？（MVCC+间隙锁）

## 三、P1 常考

### 3.1 锁

| 类型 | 用途 |
|------|------|
| 行锁 | 基于索引；无索引可能锁更多 |
| 间隙锁 | RR 下防幻 |
| 临键锁 | 记录+间隙 |
| 死锁 | 缩短事务、固定顺序；引擎回滚一方 |

### 3.2 日志

| 日志 | 作用 |
|------|------|
| redo | 持久、崩溃恢复 |
| undo | 回滚、MVCC |
| binlog | 主从、归档 |
| 两阶段提交 | redo 与 binlog 一致 |

### 3.3 慢 SQL

```text
慢日志 → explain(type/key/rows/Extra) → 索引/改写 → 验证
```

深分页：`id > last LIMIT n` 或延迟关联。

**高频题：** explain 怎么看？深分页？redo vs binlog？

### 3.4 引擎

InnoDB：事务、行锁、聚簇；默认。MyISAM：基本淘汰。
