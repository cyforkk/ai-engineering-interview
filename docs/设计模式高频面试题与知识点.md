## 二、P0 必会

### 2.1 单例 Singleton

| 点 | 结论 |
|----|------|
| 意图 | 全局唯一实例 |
| 写法 | 饿汉、DCL+volatile、静态内部类、**枚举** |
| 并发 | DCL 必须 volatile；枚举防反射/序列化 |
| Spring | 默认 Bean 单例，容器管生命周期 |

**高频题：** 有哪些写法？DCL 为何 volatile？枚举为何推荐？

### 2.2 工厂 Method / 抽象工厂 / 简单工厂

| 点 | 结论 |
|----|------|
| 简单工厂 | 一个类根据参数创建，非 GoF 正统 |
| 工厂方法 | 子类决定创建哪种产品 |
| 抽象工厂 | 产品族 |
| 场景 | 屏蔽 new、解耦实现；Spring BeanFactory |

### 2.3 代理 Proxy

| 点 | 结论 |
|----|------|
| 意图 | 控制访问、增强（日志/权限/事务） |
| 静态代理 | 手写代理类 |
| 动态 | JDK 接口代理 / CGLIB 子类 |
| Spring | AOP 核心 |

### 2.4 策略 Strategy

| 点 | 结论 |
|----|------|
| 意图 | 算法可替换，消 if-else |
| 场景 | 支付、优惠、导出格式 |
| Spring | 多实现 + `@Autowired Map` / 策略表 |

### 2.5 模板方法 Template Method

| 点 | 结论 |
|----|------|
| 意图 | 骨架固定，步骤可覆写 |
| 场景 | 导入流程、JdbcTemplate 思想 |
| 实现 | 抽象类 final 主流程 + 钩子 |

### 2.6 观察者 Observer

| 点 | 结论 |
|----|------|
| 意图 | 一改多通知，解耦 |
| 场景 | 事件、消息监听 |
| Spring | ApplicationEvent、`@EventListener` |

## 三、P1 常考

| 模式 | 一句话 | 场景 |
|------|--------|------|
| 装饰器 Decorator | 动态叠加职责 | IO 流、增强功能不改原类 |
| 适配器 Adapter | 接口转换 | 老接口兼容、第三方 SDK 包装 |
| 责任链 Chain | 请求沿链传递 | Filter、审批流、网关插件 |
| 建造者 Builder | 分步构建复杂对象 | 复杂 DTO、`StringBuilder` 思想 |

**装饰器 vs 代理：** 装饰器强调增强功能；代理强调控制访问（常隐藏真实对象）。

**适配器 vs 桥接：** 适配器补齐已有接口；桥接从设计上分离抽象与实现。

## 四、P2 速查

| 模式 | 一句话 |
|------|--------|
| 外观 Facade | 子系统统一入口 |
| 外观 vs 中介 | 外观单向简化；中介多方协调 |
| 享元 Flyweight | 共享细粒度对象 |
| 组合 Composite | 树形部分-整体 |
| 状态 State | 状态切换改变行为 |
| 命令 Command | 请求对象化，可撤销/队列 |
| 迭代器 Iterator | 统一遍历 |
| 原型 Prototype | 克隆创建 |
| 访问者 Visitor | 双分派扩展操作（少用） |

## 五、Spring 中的设计模式（必背表）

| 模式 | Spring 体现 |
|------|-------------|
| 单例 | 默认 Bean |
| 工厂 | BeanFactory、FactoryBean |
| 代理 | AOP |
| 模板方法 | JdbcTemplate、RestTemplate |
| 观察者 | 事件机制 |
| 适配器 | HandlerAdapter |
| 责任链 | 拦截器链、Filter 链 |
| 策略 | Resource 加载等 |
| 建造者 | BeanDefinitionBuilder 等 |
