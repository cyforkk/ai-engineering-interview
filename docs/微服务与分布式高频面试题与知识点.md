## 二、P0 必会

### 2.1 单体 vs 微服务

| | 单体 | 微服务 |
|--|------|--------|
| 优点 | 简单、事务好做 | 独立部署扩展、技术异构 |
| 缺点 | 膨胀难维护 | 分布式复杂、运维与一致性难 |

**何时拆：** 团队规模、模块边界清晰、独立扩展诉求。不是为了微服务而微服务。

### 2.2 CAP / BASE

| 点 | 结论 |
|----|------|
| C | 一致性 |
| A | 可用性 |
| P | 分区容忍（分布式必认 P） |
| 取舍 | CP / AP |
| BASE | 基本可用、软状态、最终一致 |

### 2.3 注册发现 / 配置 / 网关

| 能力 | 作用 | 常见 |
|------|------|------|
| 注册发现 | 服务找彼此 | Nacos / Eureka |
| 配置中心 | 动态配置 | Nacos / Config |
| 网关 | 统一入口、鉴权路由限流 | Gateway |
| 负载 | 实例间分发 | Ribbon/LoadBalancer |

### 2.4 雪崩与三大件

| 手段 | 作用 |
|------|------|
| 超时 | 防无限等 |
| 限流 | 保护自己 |
| 熔断 | 下游挂了快速失败 |
| 降级 | 保核心，次要关/默认值 |
| 隔离 | 线程池/舱壁，故障不蔓延 |

限流算法：计数器、滑动窗口、漏桶、令牌桶。

### 2.5 分布式事务

| 方案 | 点 |
|------|-----|
| 2PC/XA | 强一致，性能差 |
| 本地消息表 | 最终一致，常用 |
| 事务消息 | 半消息，如 RocketMQ |
| TCC | 尝试-确认-取消，侵入大 |
| Saga | 长事务补偿 |
| Seata | AT/TCC 等（按简历） |

**原则：** 能最终一致就别强一致。

### 2.6 幂等

唯一键、状态机、Token；MQ 消费、支付回调必谈。

### 2.7 分布式锁

Redis SET NX EX + Lua；详见 [Redis](./Redis高频面试题与知识点.md)。ZK 锁了解对比。

## 三、P1 常考

### 3.1 分布式 ID

| 方案 | 点 |
|------|-----|
| UUID | 简单无序 |
| 号段 | DB 发号 |
| 雪花 Snowflake | 趋势有序，时钟回拨问题 |
| 美团 Leaf 等 | 了解 |

### 3.2 链路追踪

TraceId 贯穿；定位慢调用；SkyWalking/Zipkin 等（按简历）。

### 3.3 一致性哈希

节点增减只影响邻近数据；虚拟节点均衡；缓存集群场景。

### 3.4 服务间通信

同步 RPC/HTTP vs 异步 MQ；超时重试要幂等。
