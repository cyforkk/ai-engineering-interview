## 二、P0 必会

### 2.1 运行时内存区域

| 区域 | 私有/共享 | 存什么 | 异常 |
|------|-----------|--------|------|
| 程序计数器 | 私有 | 字节码行号 | 规范唯一可不 OOM |
| 虚拟机栈 | 私有 | 栈帧 | SOE / OOM |
| 本地方法栈 | 私有 | native | 同上 |
| **堆** | 共享 | 对象实例 | `Java heap space` |
| 方法区/元空间 | 共享 | 类元数据等 | `Metaspace` |
| 直接内存 | 堆外 | NIO 等 | `Direct buffer memory` |

**堆分代：** Eden + 2 Survivor（约 8:1:1）+ 老年代。  
**JDK8：** 永久代 → **元空间**（本地内存）。  
**栈 vs 堆：** 栈线程私有、帧随方法；堆共享、GC 管。

**高频题**

1. 内存区域划分？哪些线程私有？  
2. 堆和栈区别？  
3. 为何元空间替代永久代？  
4. 直接内存会 OOM 吗？

### 2.2 垃圾回收

**判断存活：** 可达性分析（GC Roots）；非引用计数（循环引用）。  

**GC Roots 常考：** 栈引用、静态变量、常量、JNI、锁持有对象等。

| 算法 | 特点 | 典型 |
|------|------|------|
| 标记-清除 | 碎片 | 老年代历史 |
| 复制 | 无碎片、占空间 | 新生代 |
| 标记-整理 | 无碎片、要移动 | 老年代 |
| 分代 | 组合以上 | 现代基础 |

| GC 名 | 含义 |
|-------|------|
| Young/Minor | 新生代，Eden 满触发 |
| Full | 整堆（常含元空间），要少 |
| Mixed | G1：年轻代+部分老 Region |

**收集器选型口径：**

| 收集器 | 点 |
|--------|-----|
| Parallel | 吞吐 |
| **G1** | JDK9+ 常见默认，大堆可控停顿 |
| ZGC | 超低延迟（版本要求） |
| CMS | 历史考点；JDK14 移除 |

**高频题**

1. 如何判断对象可回收？GC Roots？  
2. 三种算法优缺点？  
3. Minor vs Full 触发？  
4. G1 是什么？和 CMS？  
5. 线上用什么 GC？为何？  

**口述：**  
> 可达性从 Roots 出发；新生代复制、老年代整理/清除；G1 按 Region 优先收垃圾多的区。

### 2.3 类加载 + 双亲委派

```text
加载 → 验证 → 准备 → 解析 → 初始化 → 使用 → 卸载
```

| 阶段 | 做什么 |
|------|--------|
| 准备 | 静态变量默认零值 |
| 初始化 | `<clinit>` 真正赋值 |
| 加载器 | Bootstrap / Ext·Platform / App / 自定义 |

**双亲委派：** 先父后子；保证核心类唯一与安全。  
**破坏例子：** 自定义 loadClass、SPI/TCCL、Tomcat 隔离。

**高频题**

1. 类加载过程？  
2. 双亲委派？为何需要？如何破坏？

### 2.4 排查路径（社招 P0）

#### OOM 类型

| 错误 | 方向 |
|------|------|
| Java heap space | dump → MAT；泄漏 vs 堆小 |
| Metaspace | 动态类/ClassLoader 泄漏 |
| Direct buffer | NIO/Netty 堆外 |
| unable to create thread | 线程过多 |
| StackOverflowError | 递归（非 OOM） |

#### 泄漏 vs 不足

- Full GC 后 Old **降不下来** → 偏泄漏  
- 峰值后能降 → 偏容量/流量  

#### CPU 100%

```text
top → top -Hp → tid 转 hex → jstack 找 nid
```

分：业务死循环 / GC 线程 / JIT。

#### 频繁 Full GC

```text
GC日志 → jstat 看回收后是否下降 → 不降则 dump
```

#### 工具

jstat、jmap、jstack、jcmd、MAT、Arthas；  
`-XX:+HeapDumpOnOutOfMemoryError`；生产慎用 live dump。

**高频题**

1. OOM 怎么查？  
2. 频繁 Full GC？  
3. CPU 飙高？  
4. 容器 OOMKilled 但无 Java OOM？（RSS 超 limit，非堆也占）

## 三、P1 常考

### 3.1 对象

| 点 | 结论 |
|----|------|
| 创建 | 类加载检查→分配(TLAB)→零值→对象头→init |
| 布局 | 头 + 实例数据 + 对齐 |
| 引用 | 强/软/弱/虚 |
| 晋升 | 年龄阈值、大对象、动态年龄、Survivor 放不下 |
| 不一定在堆 | 逃逸分析可栈上/标量替换 |

### 3.2 收集器补充

- CMS：初始标记/并发标记/重新标记/并发清除；碎片与并发失败  
- G1：Region、RSet、Mixed GC、停顿目标  
- STW / 安全点  

### 3.3 参数速记

```text
-Xms -Xmx
-XX:+UseG1GC / +UseZGC
-XX:+HeapDumpOnOutOfMemoryError
-XX:HeapDumpPath=...
GC 日志（JDK8 PrintGC* / JDK9+ -Xlog:gc*）
```

## 五、场景题速查（按出现频率）

| 序 | 场景 | 一句路径 |
|:--:|------|----------|
| 1 | 堆 OOM | 看类型→dump→MAT 引用链→泄漏or容量 |
| 2 | 内存泄漏 | GC后不降→histo/dump→GC Roots→修引用 |
| 3 | 频繁 Full GC | 日志+jstat→降不降得下→dump/调优 |
| 4 | 频繁 Young GC | 分配太猛/Eden 小→减对象/调参 |
| 5 | CPU 100% | top→线程→jstack |
| 6 | 是否 GC 卡顿 | RT 尖刺与 GC pause **时间对齐** |
| 7 | 堆外 OOM | RSS 高堆不高→Direct/Netty |
| 8 | Metaspace 涨 | 类/加载器泄漏 |
| 9 | 死锁 | jstack deadlock |
| 10 | OOMKilled | limit 含非堆，留余量 |
| 11 | 大促容量 | 压测存活集+余量+GC 目标 |
| 12 | 迁 G1/ZGC | 压测灰度看停顿与吞吐 |
| 13 | 线上 dump | 低峰/摘流，慎 live |
| 14 | 线程池堆积 | 拒绝策略+队列占堆 |
| 15 | 讲经历 | STAR：现象→工具→根因→数据→预防 |

**答题模板：** 现象 → 指标/日志 → 工具 → 根因 → 止血+根治 → 验证。
