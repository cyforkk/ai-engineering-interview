## 1. 运行时内存分区？

- **线程私有**：程序计数器、虚拟机栈、本地方法栈。
- **线程共享**：堆、方法区（JDK 8+ 元空间在本地内存）。
- **堆**：对象实例；分代：新生代（Eden + S0/S1）+ 老年代。
- **直接内存**：NIO 堆外，受本机/容器限制。

---

## 2. 如何判断对象可回收？

- **可达性分析**：从 GC Roots 出发不可达 → 可回收。
- 不是引用计数（无法很好处理循环引用）。
- GC Roots 例子：栈引用、静态变量、常量、JNI 等。

---

## 3. 常见 GC 算法 / 收集器？

- 算法：标记-清除、复制、标记-整理、分代。
- **Minor GC**：收新生代；**Full GC**：范围大、停顿通常更长。
- **G1**：Region 划分，优先回收收益高的区域，可设停顿目标（JDK 9+ 常见默认）。

---

## 4. 类加载过程？双亲委派？

- 加载 → 验证 → 准备 → 解析 → 初始化。
- **双亲委派**：先让父加载器加载，父没有再自己加载。
- **目的**：保证核心类唯一、安全。
- 可破坏：SPI、热部署、自定义 ClassLoader 等。

---

## 5. 线上排查（社招重点）

- **堆 OOM**：dump → MAT 看占用与引用链。
- **Full GC 频繁**：看回收后老年代是否下降；不降偏泄漏。
- **CPU 飙高**：找忙线程 → `jstack` 看栈。
- **容器被杀无 Java OOM**：RSS 超 cgroup，关注堆外。
