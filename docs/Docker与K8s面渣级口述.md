# Docker 与 Kubernetes · 面渣级口述（加长版）

> [知识点](./Docker与K8s高频面试题与知识点.md)

---

## 一、Docker 解决什么

镜像是分层只读模板，容器是镜像的运行实例。  
把依赖打进镜像，解决「开发能跑、生产不行」的环境漂移。  
底层是进程级隔离：Namespace 做隔离，Cgroups 做配额，比虚拟机轻、启动快。

---

## 二、为什么还要 K8s

容器一多，就要回答：跑哪台机器、挂了谁拉起、如何对外暴露、如何滚动发版、如何限制 CPU 内存。  
K8s 用声明式 API 描述期望状态，控制器负责逼近期望。

---

## 三、核心对象（务必熟）

**Pod**：最小调度单元，可多容器共享网络和卷。  
**Deployment**：声明副本数，滚动更新与回滚。  
**Service**：稳定虚拟入口，转发到后端 Pod。  
**ConfigMap / Secret**：配置与密钥。  
**Ingress**：七层 HTTP 路由入口（常见组合）。

**探针：**  
liveness 失败重启容器；readiness 失败摘除流量。探针接口要轻，别每次打重逻辑。

**资源：**  
request 影响调度；limit 是上限。Java 特别要注意：容器 limit 限制的是进程 RSS，堆外、直接内存、元空间都算。只调 -Xmx 仍可能被 OOMKill，且 JVM 不一定抛 Java OOM。

---

## 四、发布与排障

滚动更新：新 Pod Ready 再缩旧 Pod。  
常见故障：ImagePullBackOff、CrashLoopBackOff、OOMKilled、配置没挂载、探针配错。  
排障顺序：事件 → 日志 → describe → 资源使用 → 最近发布。

---

## 五、收尾

后端面试够用：镜像/容器、Pod/Deployment/Service、探针、request/limit、滚动发布；结合一次真实上线或回滚经历即可。

## 面试临场补充（再加 1 分钟）

### 如果面试官说「再展开一点」

我会补三点：第一，关键参数或配置叫什么、大概怎么设；第二，失败时系统如何表现、用户侧如何降级；第三，我用什么指标判断方案有效，比如延迟、错误率、超卖次数、评测分数。

### 如果面试官问「有没有坑」

任何缓存、MQ、容器、模型方案都有坑。我会主动说一个真实或典型坑，以及怎么发现、怎么修、怎么防再发生。这样比只背正确步骤更像干过活。

### 和项目怎么挂钩

即使项目不是标准秒杀或标准 Agent，也可以迁移：限流、幂等、异步、检索、配置版本、发布回滚，都是同一套肌肉。面试前把简历里的两点业务，映射到本专题的两个问题上。
