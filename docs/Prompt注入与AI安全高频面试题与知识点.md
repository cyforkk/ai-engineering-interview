## 一、攻击面在哪里

LLM 应用把**不可信文本**与**系统指令、工具权限**放在同一上下文。攻击者通过用户输入或「间接注入」（网页/邮件/上传文档被检索进来）试图：

- 覆盖系统目标（忽略以上指令…）  
- 套取系统提示、密钥、他人数据  
- 诱使模型调用危险工具（转账、删库、外发）

**所以然：** 模型会「听话」地跟从上下文中的指令式文本；不能把安全只押在「模型懂规矩」。

---

## 二、为什么「提示里写禁止」不够

提示是软约束，可被更强诱导、编码绕过、多轮渗透削弱。必须**系统层硬控制**：

| 层 | 手段 |
|----|------|
| 权限 | 工具最小权限；默认只读 |
| 执行 | 写操作 HITL；参数白名单 |
| 数据 | 租户隔离；脱敏；出域审批 |
| 检测 | 输入输出过滤、异常工具调用告警 |
| 运维 | 审计日志、一键熔断 |

---

## 三、间接注入为什么更难防

内容来自检索，像「正常知识」进入上下文。防御：来源标记、不信任文档中的指令语气、工具调用不因文档要求而提权、关键操作二次确认。
