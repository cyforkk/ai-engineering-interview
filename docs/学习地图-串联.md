## 跨线逻辑依赖（为什么这样排）

| 先学 | 后学 | 原因 |
|------|------|------|
| LLM 基础 | Prompt / RAG | 懂 token/幻觉再谈提示与检索 |
| AI 应用工程 | 向量库 / Agent | 先有架构再下钻零件 |
| 评测 SOP | 发版/LLMOps | 没有标准无法谈回归 |
| 安全 | Agent 工具 | 工具权限建立在威胁模型上 |
| Java 集合 | 并发 / JVM | 对象与容器是后续基础 |
| MySQL | Redis 一致性 | 先库后缓存双写 |
| Spring | 微服务 | 单服务扎实再拆 |
| 网络 | OS epoll / K8s | 连接模型与部署相关 |
| MQ 通用 | Kafka/RMQ | 先共性再产品 |
| 系统设计 | STAR | 设计能力落到项目故事 |
