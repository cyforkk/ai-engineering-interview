## 1. 进程和线程

- **进程：** 一个独立小程序世界，有自己的内存地盘（隔离好）  
- **线程：** 世界里的多个工人，**共用仓库**（堆），所以要排队、加锁  

Java 里多线程一起改同一个 HashMap 会出事，就是因为**共享堆**。

---

## 2. 用户态 / 内核态

普通程序不能随便碰硬件。  
要读文件、上网，得喊操作系统帮忙（系统调用），像办事要去柜台。  
跑柜台太勤也慢，所以要有缓冲、批量、epoll 这些优化。

---

## 3. epoll（后端常问）

几千上万连接时，不能一人一个服务员死等。  
epoll：一个调度员盯着，**谁就绪就招呼谁**。  
Nginx、Redis、Netty 都是这类思路。

---

## 4. 死锁

四人过独木桥互相卡住：你等我、我等你。  
四个条件都满足才容易死锁；破坏一个就能预防（比如大家按固定顺序拿锁）。
