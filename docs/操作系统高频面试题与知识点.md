## 1. 进程 vs 线程？

- **进程**：资源分配单位，独立地址空间，隔离强。
- **线程**：调度单位，同进程共享堆/全局，需同步。
- Java 线程共享堆 → 要考虑线程安全。

---

## 2. 用户态 / 内核态？系统调用？

- 用户程序不能直接访问硬件 → 通过系统调用进内核。
- 切换有成本 → 缓冲、批量、多路复用、零拷贝。

---

## 3. 上下文切换？

- 保存/恢复寄存器等运行现场；过多切换浪费 CPU。

---

## 4. epoll？

- IO 多路复用：一线程监听多连接，处理就绪连接。
- 相对 select/poll：更适合海量连接（就绪队列）。
- Nginx / Redis / Netty 等使用同类思想。

---

## 5. 死锁？

- 四条件：互斥、占有且等待、不可抢占、循环等待。
- 预防：固定加锁顺序、超时、减少锁粒度等。

---

## 6. 虚拟内存？

- 页表映射虚拟地址到物理内存；缺页慢。
- 容器还受 cgroup 限制。
