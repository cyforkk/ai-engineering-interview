# 一、超高频（几乎必问）

## 1. 进程与线程

### 1.1 进程 vs 线程？

| | 进程 | 线程 |
|--|------|------|
| 单位 | **资源分配** | **CPU 调度** |
| 地址空间 | 独立 | 同进程共享堆、全局等 |
| 私有 | 页表、资源 | 栈、寄存器、PC（及 TCB） |
| 通信 | IPC | 共享内存（需同步） |
| 切换 | 更重 | 相对轻 |

### 1.2 为什么有了进程还要线程？

- 线程创建/切换更轻  
- 共享地址空间，协作方便  
- 多核可并行执行同进程内多线程  
- 阻塞时其他线程仍可推进（同进程内）  

### 1.3 进程状态与转换？

```text
创建 → 就绪 ⇄ 运行 → 阻塞 → 就绪
                ↘ 终止
```

| 状态 | 含义 |
|------|------|
| 创建/新建 | 正在创建 |
| 就绪 Ready | 可运行，等 CPU |
| 运行 Running | 占 CPU |
| 阻塞 Blocked | 等 IO/事件 |
| 终止 | 结束 |

- Java 六状态是语言层映射，见 [并发卷](./并发高频面试题与知识点.md)。  

### 1.4 上下文切换？进程 vs 线程开销？

- 切换：保存/恢复寄存器、PC、栈等；可能刷新 TLB/缓存  
- **进程切换**还要切地址空间/页表 → 更贵  
- **线程切换**同进程内主要换寄存器/栈 → 相对便宜  
- 过多切换 → CPU 空转在切换上  

### 1.5 PCB / TCB？

| | 含什么（直觉） |
|--|----------------|
| **PCB** | PID、状态、寄存器现场、页表/内存信息、打开文件、调度信息… |
| **TCB** | 线程 ID、寄存器、栈指针、优先级、所属进程… |

- 内核靠 PCB/TCB 管理调度与切换。  

### 1.6 IPC 方式？

| 方式 | 优点 | 缺点 |
|------|------|------|
| 管道 pipe | 简单 | 血缘/单向常见限制 |
| 消息队列 | 格式消息 | 拷贝、大小限制 |
| **共享内存** | 最快 | 要自己同步 |
| 信号 signal | 异步通知 | 信息量少 |
| Socket | 可跨机 | 相对重 |
| 信号量 | 同步 | 编程复杂 |

### 1.7 协程？

- 用户态轻量「线程」，由用户/运行时调度  
- 切换不经内核（一般），开销更小  
- vs 线程：无独立内核调度实体（实现多样）；Java 虚拟线程见并发卷

## 2. 用户态与内核态

### 2.1 为何区分？

- **保护**：用户程序不能随意碰硬件与内核数据  
- 用户态权限低；内核态可执行特权指令  

### 2.2 如何切换？

| 途径 | 说明 |
|------|------|
| **系统调用** | 主动请求内核服务（read/write/socket…） |
| **中断** | 硬件/时钟打断 |
| **异常** | 缺页、除零等 |

### 2.3 系统调用过程（简化）

```text
用户调库函数 → trap 陷入内核
 → 保存用户态现场
 → 内核执行服务
 → 恢复现场 → 返回用户态
```

- 有切换成本 → 缓冲、批量、mmap、epoll 减少切换  

---

## 3. 死锁

### 3.1 定义 + 四必要条件

- 多进程/线程互相等待对方持有的资源，永久阻塞  

1. **互斥**  
2. **占有且等待**  
3. **不可抢占**  
4. **循环等待**  

### 3.2 预防 / 避免

| 策略 | 做法 |
|------|------|
| 预防 | 破坏四条件之一（顺序加锁、一次申请、可超时释放） |
| 避免 | **银行家算法**等：分配前检查是否进入不安全状态 |

### 3.3 检测与解除

- 检测：资源分配图找环；银行家安全序列  
- 解除：剥夺资源、回滚、杀进程  
- Java：`jstack` 报告 deadlocks  

### 3.4 死锁代码直觉（Java）

```text
线程A: lock1 → lock2
线程B: lock2 → lock1
```

- 固定加锁顺序可破环。完整示例可手写两个 synchronized 交叉。

## 4. 内存管理（核心）

### 4.1 虚拟内存？作用？

- 进程看到连续虚拟地址；映射到物理页/磁盘  
- **作用**：隔离、可超卖物理内存、保护、方便加载  

### 4.2 分页 vs 分段？段页式？

| | 分页 | 分段 |
|--|------|------|
| 单位 | 固定页 | 逻辑段（变长） |
| 碎片 | 内碎片 | 外碎片 |
| 目的 | 管理物理内存 | 逻辑模块（代码/数据） |

- **段页式**：先分段再分页，兼得逻辑与固定页管理  

### 4.3 页表？多级页表？

- 页表：虚拟页号 → 物理页框号 + 权限/存在位  
- **多级页表**：不建完整巨大一级表；只分配用到的中间页表 → **省内存**（稀疏地址空间）  

### 4.4 TLB？

- Translation Lookaside Buffer：**页表缓存**  
- 命中则免查内存页表，加速地址翻译  

### 4.5 缺页中断流程？

```text
访问页不在内存 → 缺页异常
 → 内核找/换出页框
 → 从磁盘调入页
 → 更新页表
 → 重新执行指令
```

### 4.6 页面置换算法？

| 算法 | 特点 |
|------|------|
| FIFO | 简单；可能 Belady 异常 |
| **LRU** | 近似最优；实现成本高 |
| Clock | LRU 近似，实用 |
| LFU | 按频率；实现复杂 |

### 4.7 写时复制 COW？

- fork 后父子共享物理页，标只读  
- 任一方**写**时再复制页  
- **作用**：加快 fork，省内存（直到写入）

# 二、高频

## 5. 进程调度

| 算法 | 特点 |
|------|------|
| FCFS | 公平简单；短作业等长作业 |
| SJF | 短作业优先；可能饥饿 |
| 时间片轮转 RR | 交互公平 |
| 优先级 | 重要先跑；防饥饿要老化 |
| 多级反馈队列 | 综合实用 |

- **抢占** vs **非抢占**：能否被打断夺走 CPU  
- **饥饿**：低优先级长期不运行 → 老化提高优先级  

## 6. 同步与互斥

| 概念 | 含义 |
|------|------|
| 互斥 | 临界区同时只有一个 |
| 同步 | 协作次序（先后关系） |
| 信号量 | P/V 操作；可实现互斥与同步 |
| 自旋锁 | 忙等；短临界区、多核 |
| 互斥锁 | 阻塞睡眠；长临界区 |
| 管程 | 语言级封装共享+条件变量 |

- 与 Java `synchronized`/Lock/Semaphore 对应见并发卷  

## 7. IO 模型

| 模型 | 特点 |
|------|------|
| 阻塞 BIO | 一连接一线程易 |
| 非阻塞 | 轮询 |
| **IO 多路复用** | select/poll/epoll 一线程多连接 |
| 信号驱动 | 信号通知可读 |
| 异步 AIO | 内核完成后通知 |

### select / poll / epoll

| | select/poll | epoll |
|--|-------------|-------|
| 效率 | 扫描全部 fd | 就绪队列，适合海量 |
| 数量 | select 有上限 | 很大 |
| 触发 | — | **LT 水平** / **ET 边缘** |

- **LT**：有数据就通知，可只读部分  
- **ET**：状态变化通知，需一次读干净，更高效也更易错  
- Redis/Nginx/Netty 用 epoll 思想  

### 零拷贝

- 减少用户态/内核态数据拷贝次数  
- 实现：`sendfile`、`mmap`+write、splice 等  
- 场景：静态文件发送、Kafka 等  

## 8. 其他高频

### 中断

- 硬中断（设备）/ 软中断；保存现场 → 处理 → 恢复  

### fork / exec / wait

| 调用 | 作用 |
|------|------|
| fork | 复制进程（COW） |
| exec | 用新程序覆盖地址空间 |
| wait | 父等子结束，收尸 |

### 僵尸 vs 孤儿

| | 僵尸 Zombie | 孤儿 Orphan |
|--|-------------|-------------|
| 含义 | 子结束父未 wait，PCB 残留 | 父先死，子托孤 init |
| 避免僵尸 | wait/waitpid；或 signal 处理 SIGCHLD | 系统接管 |

# 三、中频

## 9. 文件系统

| 点 | 要点 |
|----|------|
| 硬链接 | 同 inode，不能跨文件系统、目录通常不硬链 |
| 软链接 | 路径指针，可跨、可悬空 |
| inode | 文件元数据+数据块指针 |
| dentry | 目录项：名字→inode |
| 打开文件 | 路径解析 → 权限 → 文件描述符表 |

### 磁盘调度

FCFS、SSTF（近）、SCAN/C-SCAN（电梯）  

## 10. 内存进阶

| 概念 | 要点 |
|------|------|
| 内碎片 | 页内浪费 |
| 外碎片 | 空闲不连续 |
| 伙伴系统 | 按 2 的幂块分配，减外碎片 |
| malloc | 小 brk/sbrk；大 mmap |
| Swap | 换出到磁盘；**抖动** = 频繁换入换出 |
| OOM Killer | 内存耗尽杀进程保系统 |

### Linux 常用命令

- `ps`/`top`/`htop` 进程  
- `free`/`vmstat` 内存  
- `cat /proc/pid/status`  

### 缓冲 vs 缓存

- 缓冲 Buffer：缓和速度差（写合并）  
- 缓存 Cache：加速重复读
