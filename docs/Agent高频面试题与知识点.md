## 二、P0 必会

### 2.1 是什么 / 对比

| 概念 | 一句话 |
|------|--------|
| Chat | 生成回复 |
| RAG | 检索再生成 |
| Workflow | 固定步骤 |
| **Agent** | 动态规划+工具+观察循环直到完成 |
| Fine-tune | 改权重，不替代检索/工具 |

**高频题：** Agent 是什么？和 RAG/工作流区别？

---

### 2.2 ReAct（第一范式）

```text
Thought → Action(tool) → Observation → … → Final Answer
```

**防死循环（必背）：** max_steps、重复检测、超时、工具错误熔断、HITL、改 Plan-Execute/工作流。

**高频题：** ReAct 流程？会死循环吗？怎么防？

---

### 2.3 工具调用

1. 提供 Schema（名、描述、参数）  
2. 模型输出 structured call  
3. 运行时执行，结果回填  
4. 继续推理或结束  

**设计：** 描述清晰、参数严、错误结构化、写操作审批、allowlist、审计。

**高频题：** Function Calling 流程？如何防乱调删库？

## 三、P1 常考

### 3.1 记忆

| 类型 | 手段 |
|------|------|
| 短时 | 窗口、本轮轨迹 |
| 长时 | 向量/KV/画像 |
| 压缩 | 摘要 + 检索相关片段 |

### 3.2 RAG × Agent

- 检索常挂成 `search_kb` 工具  
- 分块、混合检索、重排、强制引用、无据拒答  
- 评估：忠实度、相关性、任务完成率  

### 3.3 Plan-and-Execute

先全局计划再执行；长任务更稳；环境变要 replan。  
短任务 ReAct；复杂常混合。

### 3.4 评估与观测

任务完成率、工具正确率、步数、费用、人工接管率；全链路 trace。

## 四、P2 加分

| 点 | 一句话 |
|----|--------|
| 多 Agent | Supervisor/流水线/辩论；有总控与轮次上限 |
| MCP | 工具/资源互联协议；FC 是模型侧调用格式 |
| 框架 | LangGraph 偏可控状态图；按简历深挖 |
| 安全 | Prompt 注入、越权、沙箱、最小权限 |
| 成本 | 小模型路由、缓存、预算熔断 |

## 五、系统设计答题序（场景）

```text
目标边界 → 架构(编排/工具/RAG/记忆) → 规划策略
→ 可靠(步数/重试/HITL) → 安全 → 评估与成本 → 演进
```

例：企业知识库 Agent、客服+转人工、SQL Agent。
