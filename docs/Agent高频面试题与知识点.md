## 1. Agent 到底是啥？

### 先记一句

普通聊天：你问它答。  
**Agent：有目标，会多步想，会动手用工具，看结果再接着干。**

### 生活例子

你让助理「帮我订下周去上海的机票」：

1. 先查日历（工具）  
2. 再比价（工具）  
3. 发现要护照信息，回来问你  
4. 最后下单（危险操作要你确认）

这就是 Agent 味道：**不是一次说完，而是边做边看**。

公式：`大脑(LLM) + 计划 + 记忆 + 工具 + 刹车（护栏）`

---

## 2. 和 Chat / RAG / 工作流差在哪？

| | 像什么 | 特点 |
|--|--------|------|
| Chat | 对面聊天 | 只说话 |
| RAG | 先翻资料再答 | 知识可更新 |
| 工作流 | 工厂流水线 | 步骤人写死，稳 |
| Agent | 外包全能助理 | 灵活，但贵、难控 |

**啥时候别上 Agent？**  
步骤固定、合规要求严、特别在意成本和可控——用工作流更合适。

---

## 3. ReAct：想一下 → 做一下 → 看结果

```text
思考 → 调用工具 → 观察返回 → 再思考 … → 最终回答
```

### 为啥要刹车？

助理可能：

- 同一件事问十遍  
- 工具一直失败还死磕  
- 步数爆炸，钱烧没了  

所以必须有：

1. 最多几步  
2. 重复动作检测  
3. 超时  
4. 工具失败就停  
5. 转账/删数据等人批准（HITL）

---

## 4. 工具谁来执行？

模型只负责说：「我想调用查天气(城市=上海)」。  
**真正执行的是你的后端**：验权限、验参数、记日志。  
千万别信模型说「我有权限」。
