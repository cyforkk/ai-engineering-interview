## 一、要搞定什么

Agent ≠ 多聊两句。它是：**目标驱动 → 推理 → 调工具 → 看结果 → 再决策**，直到完成或停止。

公式：**Agent ≈ LLM + 规划 + 记忆 + 工具**

**主教材：** [Agent面渣级口述.md](./Agent面渣级口述.md)

## 二、和 Chat / RAG / 工作流（P0）

| 概念 | 白话 |
|------|------|
| Chat | 主要生成回复 |
| RAG | 先检索再生成（可当工具） |
| 工作流 | 步骤人写死，可控 |
| Agent | 步骤模型动态选，灵活但贵、难控 |

### 面试怎么答

> 多了工具与多步循环；比工作流灵活，比纯聊天能行动；工程要限步、权限、人机确认。

## 三、ReAct（P0）

### 白话

Thought（想）→ Action（调工具）→ Observation（看结果）→ 再想… → Final Answer。  
容易死循环：重复调工具、步数爆炸。

**必做限制：** max_steps、重复检测、超时、工具错误熔断、危险操作 HITL。

**完整稿：** [Agent面渣 · ReAct](./Agent面渣级口述.md)

---

## 四、工具调用（P0）

Schema 描述工具 → 模型输出结构化调用 → 服务端**真执行** → 结果回填。  
描述要清楚；参数校验；写操作审批；不信模型填的权限。
