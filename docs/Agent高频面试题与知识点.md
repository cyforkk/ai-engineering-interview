## 一、本专题要搞定什么

面试官问 Agent，是在考：

1. 你是否分得清 **Chat / RAG / 工作流 / Agent**  
2. 能否讲清 **ReAct 循环** 与 **死循环怎么防**  
3. 工具调用的工程边界（权限、校验、HITL）  
4. 何时 **不该** 上 Agent（成本、可控性）

## 二、Agent 是什么（P0）

### 白话

普通聊天：模型根据上下文 **生成一段回复**。  
**Agent**：有一个目标，模型会 **多步推理**，中间可以 **调用工具**（搜库、查 API、写文件、跑代码），观察结果后再决定下一步，直到完成或触发停止条件。

公式（面试够用）：

```text
Agent ≈ LLM（大脑）+ 规划 + 记忆 + 工具（手）+ 停止/护栏
```

### 面试怎么答（约 1 分钟，可照着说）

> 我认为 Agent 不是多聊几句，而是目标驱动的闭环：模型根据当前状态决定下一步，可以调用工具获取信息或执行动作，把观察结果写回上下文，再继续推理，直到达成目标或命中停止条件。  
> 工程上关键不只是 prompt，而是工具权限、最大步数、超时、审计和失败降级。  
> 和纯 Chat 比它能行动；和固定工作流比它更灵活，但更贵、更难控，所以要护栏。

## 三、Chat / RAG / 工作流 / Agent 对比（P0）

| 概念 | 白话 | 可控性 | 成本 |
|------|------|--------|------|
| Chat | 一问一答生成 | 高 | 低 |
| RAG | 先检索再生成，知识可更新 | 中高 | 中 |
| 工作流 | 步骤人写死（DAG） | **最高** | 中 |
| Agent | 步骤模型动态选 | 较低 | **高** |

### 面试怎么答

> RAG 解决的是「知识从哪来、怎么 grounding」；工作流是确定性编排；Agent 是把「下一步干什么」交给模型。  
> 生产里常混合：外层工作流控风险，内层局部允许 Agent 调工具；检索本身也可以是 Agent 的一个 tool。

### 追问：什么时候不用 Agent？

> 步骤固定、合规要求强、延迟/成本敏感、工具副作用大——优先工作流 + RAG，而不是开放式 Agent。

## 四、ReAct（P0）

### 白话

经典模式：

```text
Thought（推理下一步）
→ Action（选工具 + 参数）
→ Observation（工具返回）
→ 再 Thought …
→ Final Answer
```

问题：模型可能 **重复调用同一工具**、步数爆炸、工具报错后胡乱重试 → **死循环或账单爆炸**。

### 面试怎么答（约 1.5 分钟）

> ReAct 把推理和行动交错：先 Thought，再 Action，拿到 Observation 再继续。  
> 好处是可解释、能接入真实工具。风险是循环与成本。  
> 我会做：max_steps 硬限制、相同 Action 指纹去重、全局超时、工具错误次数熔断、危险写操作必须 Human-in-the-loop、每步 trace 落日志方便复盘。  
> 达不成目标就降级：返回已收集证据、或转人工，而不是无限转。

### 防死循环清单（背这 5 条）

1. `max_steps`  
2. 重复 Action 检测  
3. 超时  
4. 工具错误熔断  
5. 危险操作审批（HITL）

## 五、工具调用（Function Calling）（P0）

### 白话

1. 你用 JSON Schema 描述工具（名字、参数、含义）  
2. 模型 **输出结构化调用意图**（不是它真的碰到了你的数据库）  
3. **你的后端** 校验参数、检查权限、真正执行  
4. 把结果作为 Observation 塞回模型  

**安全原则：** 永不信任模型填的「我有权限」；写操作默认要审批；参数白名单；超时与重试策略由你定。

### 面试怎么答

> Function Calling 是模型生成调用计划，执行权在服务端。  
> 我会强调 schema 清晰、参数校验、鉴权在服务端、副作用工具分级、全程审计日志。  
> 工具描述含糊会导致乱调；返回结果过长要摘要，避免撑爆上下文。

## 六、记忆（P1）

| 类型 | 白话 | 注意 |
|------|------|------|
| 短时 | 当前对话窗口 | 有 token 上限 |
| 长时 | 向量库/画像/DB | 隐私与过期 |
| 工作记忆 | 本轮 Agent 中间状态 | 步数多要压缩 |

过长历史：**摘要** 后再塞，不要无限 append。

### 面试怎么答

> 短时靠上下文，长时靠外部存储检索。  
> 工程上要做摘要、检索过滤和权限隔离，防止把 A 用户记忆检索给 B。
