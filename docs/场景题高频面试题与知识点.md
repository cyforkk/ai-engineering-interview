> 场景题（项目 + 线上问题 + 高并发设计）比重常 **超过纯八股**。  
> 考察：用原理 **解决实际问题** 的分层能力与权衡。

### 标准回答框架（强烈推荐）

```text
1. 澄清问题（QPS、数据量、一致性、是否允许最终一致）
2. 分析核心难点（超卖、热点、一致性、可用性…）
3. 分层方案（前端 → 网关 → 服务 → 缓存 → DB → MQ）
4. 对比方案 + 选型理由
5. 兜底与监控（降级、告警、对账、补偿）
6. 结合自己项目经验（最加分）
```

# 一、超高频（几乎必问）

## 1. 高并发秒杀 / 抢购

> 📋 **映射：** 勾「秒杀/库存」「高并发/限流」· 钩子：我在【】用限流+预扣+… · [样例Java](./项目映射样例-Java.md)

### 1.1 如何设计秒杀？

| 层 | 手段 |
|----|------|
| 前端 | 防抖、验证、静态化、CDN |
| 网关 | 限流、黑名单 |
| 服务 | 预扣库存、幂等、异步下单 |
| 缓存 | Redis 预热 + Lua 原子扣 |
| MQ | 削峰创建订单 |
| DB | 少写；条件更新兜底 |

详见：[系统设计·秒杀](./后端系统设计高频面试题与知识点.md)

### 1.2 防超卖？

- Redis：`DECR` / Lua 判断 >0 再扣  
- DB：`UPDATE ... SET stock=stock-1 WHERE stock>0`  
- 订单唯一 + 对账回补  

### 1.3 防重复下单？

- 用户+活动 幂等 token / 唯一索引  
- 前端防重复提交 + 服务端防重表  

### 1.4 热点库存扣减？

| 方案 | 说明 |
|------|------|
| Redis + Lua | 原子、扛并发 |
| 分段库存 | 减热点 |
| DB 乐观锁 | 兜底一致性 |

### 1.5 分层限流？

- 前端限点 → 网关令牌桶 → 服务信号量/Sentinel → 依赖隔离

## 2. 缓存相关场景

> 📋 **映射：** 勾「缓存三连/双写一致」· 钩子：穿透/击穿/雪崩我分别用【】· 双写顺序【】

### 2.1 穿透 / 击穿 / 雪崩（综合）

| 问题 | 方案组合 |
|------|----------|
| 穿透 | 空值缓存 + 布隆 + 校验 |
| 击穿 | 互斥重建 + 逻辑过期 |
| 雪崩 | TTL 随机 + Redis HA + 限流降级 + 预热 |

完整：[Redis·三连](./Redis高频面试题与知识点.md)

### 2.2 双写一致性？

| 策略 | 说明 |
|------|------|
| Cache Aside | **先 DB 再删缓存**（主流） |
| 延迟双删 | 删→更DB→延迟再删 |
| binlog 订阅 | 异步删/更新，最终一致 |

- 强一致成本高；业务多接受最终一致 + TTL  

### 2.3 热点 Key？

- 本地缓存（Caffeine）+ Redis  
- 互斥重建；逻辑过期  
- key 打散分片计数再汇总  

### 2.4 大 Key / 热 Key 治理？

| | 发现 | 治理 |
|--|------|------|
| 大 Key | bigkeys、监控、慢日志 | 拆分、压缩、UNLINK |
| 热 Key | 代理统计、热点探测 | 本地缓存、限流、读写分离 |

## 3. 分布式锁

> 📋 **映射：** 勾「分布式锁」· 钩子：锁实现【Redis/…】防误解【value+Lua】· 业务场景【】

### 3.1 可靠实现？

```text
SET key unique NX EX
解锁：Lua 校验 value 再 del
```

### 3.2 Redisson 看门狗？

- 业务未完自动 **续期**，防锁过期  
- 可重入：线程标识 + 重入计数  

### 3.3 常见问题？

| 问题 | 处理 |
|------|------|
| 误删 | 唯一 value + Lua |
| 过期 | 续期/合理 TTL |
| 主从丢锁 | 接受风险或 ZK/etcd；知 RedLock 争议 |

完整：[Redis·锁](./Redis高频面试题与知识点.md)

## 4. 限流与防刷

> 📋 **映射：** 勾「高并发/限流」· 钩子：网关+服务两层，算法【令牌桶/…】，超限表现【】

### 4.1 接口限流设计？

- 算法：固定/滑动窗口、令牌桶、漏桶  
- 单机：Guava、信号量  
- 分布式：Redis + Lua、网关  

### 4.2 防刷？

- 登录态、验证码、设备指纹  
- UID/IP 限流、黑名单  
- 业务 token 一次性  

### 4.3 密码错误次数？

- Redis 计数 + TTL（如 15 分钟）  
- 超阈值锁定 + 验证码  
- 审计日志  

详见：[系统设计·限流](./后端系统设计高频面试题与知识点.md)

> 📋 **映射：** 勾「MQ 不丢不重/积压」· 钩子：丢/重/积压在项目里对应【确认/幂等/扩容】

## 7. 线上问题排查

> 📋 **映射：** 勾「JVM 排查」· 钩子：一次线上【现象→工具→根因→止血】· [样例可借鉴排查表述](./项目映射样例-Java.md)

### 7.1 CPU 飙高

```text
top -H 找忙线程 → 线程 id 转 16 进制
 → jstack 对栈
 → 看死循环/正则/序列化/GC/锁竞争
 → 限流止血 + 修代码
```

### 7.2 内存泄漏 / OOM

```text
监控堆/元空间 → dump（HeapDumpOnOOM）
 → MAT 看 Dominator/引用链
 → 修泄漏或调参
```

### 7.3 频繁 Full GC

- jstat/GC 日志：回收后老年代是否下降  
- 不降 → 泄漏；能降又涨 → 分配过快/晋升  
- 工具：MAT、Arthas  

完整：[JVM·排查](./JVM高频面试题与知识点.md) · [面渣场景](./JVM面渣级口述.md)

### 7.4 接口突然变慢

```text
确认范围（单接口/全站）
 → 链路追踪（耗时 span）
 → 慢 SQL / Redis / 下游 RT
 → 锁竞争 / 线程池满 / GC 停顿
 → 限流降级止血
```

### 7.5 死锁

- `jstack` / `SHOW ENGINE INNODB STATUS`  
- 固定加锁顺序、缩短事务、tryLock
