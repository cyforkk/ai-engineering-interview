# 一、超高频～高频

## 1. 倒排索引原理？

- 正排：文档 → 词  
- **倒排**：词 → 文档列表（+ 位置/词频等）  
- 适合全文检索、关键词快速找文档  

## 2. 文档写入流程（简化）

```text
写入 → 内存 buffer
     → refresh 到 OS 文件（可搜，默认约 1s）
     → translog 保证持久
     → flush 落盘 lucene commit，清空 translog 段
```

- **refresh**：近实时可搜  
- **flush**：持久化  
- **translog**：防丢  

## 3. 分片与副本？

| | 主分片 primary | 副本 replica |
|--|----------------|--------------|
| 作用 | 数据分片扩容 | 高可用、读扩展 |
| 写入 | 先写主 | 主成功后同步副本 |

- 索引创建后 **主分片数不能改**（传统）；副本可改。  

## 4. 如何保证写入一致性直觉？

- 写主分片成功（按配置 wait_for_active_shards）再返回  
- 副本异步/同步策略影响延迟与安全  

## 5. 深度分页？

- `from+size` 深分页代价大（每分片取再归并）  
- 方案：  
  - **search_after**（推荐实时翻页）  
  - **scroll**（快照遍历，适合导出）  

## 6. ES vs MySQL 边界？

| 用 MySQL | 用 ES |
|----------|-------|
| 事务、强一致、复杂关联 | 全文检索、多条件筛选、日志检索、相关性排序 |
| 订单核心状态 | 商品搜索、内容搜索 |

- 常见：MySQL 主存 + 同步/双写 ES 做搜
