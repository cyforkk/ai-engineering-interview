## 1. 并发要解决什么？

- **原子性**：多步操作不被打断成中间态。
- **可见性**：一个线程的写，其他线程能看到。
- **有序性**：禁止非法重排序，保证多线程语义。

---

## 2. synchronized 和 Lock 区别？

- **synchronized**：JVM 关键字；自动加锁释放；可重入；非公平。
- **Lock（如 ReentrantLock）**：API 锁；可尝试锁、可中断、可超时、可公平；需手动 unlock（常 `try/finally`）；可多 Condition。

---

## 3. volatile 作用？能保证 i++ 吗？

- 作用：**可见性** + **有序性**（禁止部分重排序）。
- **不能**保证复合操作原子性（`i++` 是读改写）。
- 适用：状态标志、一个写多个读的简单发布等。

---

## 4. 线程池核心参数与流程？

- 参数：corePoolSize、maximumPoolSize、keepAliveTime、workQueue、threadFactory、handler。
- 提交流程：
  1. 线程数 < core → 创建核心线程。  
  2. ≥ core → 入队。  
  3. 队列满且 < max → 创建非核心线程。  
  4. 仍满 → 拒绝策略。  
- 拒绝策略：Abort、CallerRuns、Discard、DiscardOldest。
- **坑**：无界队列可能导致 max 线程用不上、任务堆到 OOM。

---

## 5. ThreadLocal 注意点？

- 每个线程一份变量副本。
- 线程池复用线程 → 用完必须 `remove`，防脏数据与内存泄漏。

---

## 6. CAS？ABA？

- CAS：比较并交换，无锁更新；失败重试。
- ABA：值 A→B→A，CAS 误判未变 → 版本号 / `AtomicStampedReference`。
- `AtomicInteger` 等基于 CAS。

---

## 7. AQS 一句话？

- 同步器框架：`state` + 等待队列；`ReentrantLock`、`CountDownLatch` 等基于它。
