## 一、本专题要搞定什么

并发面试几乎总围着四件事转：

1. **线程池**怎么建、任务怎么走（最高频）  
2. **synchronized / volatile / CAS** 各解决什么  
3. **安全发布、可见性**懂不懂  
4. 有没有踩过 **ThreadLocal / 死锁**  

---

## 二、线程池（P0 第一优先级）

### 白话

不要每次 `new Thread`：创建销毁贵、还可能把机器打满。  
线程池 = **工人班组**：核心工人先干活；忙不过来就任务排队；队满了再招临时工；临时工也满了就按规则拒绝。

**七大参数（名字要能对上号）：**

| 参数 | 人话 |
|------|------|
| corePoolSize | 常驻工人数 |
| maximumPoolSize | 含临时工的上限 |
| keepAliveTime | 临时工闲多久解雇 |
| workQueue | 任务排队的地方 |
| threadFactory | 怎么造线程（可起名） |
| handler | 队满且人满时怎么办 |

**提交顺序（必须按序说，背错会翻车）：**

```text
① 当前线程数 < core     → 新建核心线程执行
② 否则                  → 放进队列
③ 队列满且 < max        → 新建非核心线程
④ 还不行                → 拒绝策略
```

注意：若队列是**无界**的，队列几乎不满，**很难走到③**，max 形同虚设，任务堆积可能 OOM。生产用**有界队列**。

**拒绝策略：** Abort 抛异常（常见）；CallerRuns 调用者自己跑（背压）；Discard / DiscardOldest 丢任务要非常谨慎。

**别用 Executors 快捷工厂：** Fixed 默认无界队列；Cached 线程数可爆。阿里规范也建议自己 `new ThreadPoolExecutor`。

### 面试怎么答（2 分钟结构）

> 生产用 ThreadPoolExecutor 自建。  
> 说清七参数 → 严格按 核心→队列→最大→拒绝 的流程。  
> 有界队列 + 明确拒绝 + 按业务隔离池。  
> 线程数：CPU 密约 N+1；IO 密可更大，最终压测。

### 追问

| 问 | 答 |
|----|-----|
| 核心线程会超时回收吗？ | 默认不会；allowCoreThreadTimeOut 可开 |
| 队列选什么？ | 一般有界 ArrayBlockingQueue 等 |
| 如何观测？ | 活跃数、队列长度、拒绝次数 |

**完整稿：** [并发面渣 · 线程池长答](./并发面渣级口述.md)

## 三、synchronized

### 白话

同一时刻只让一个线程进临界区，保证原子性；退出锁时修改对其它线程可见。  
可以锁方法或代码块；锁的是对象（实例方法锁 this，静态锁 Class）。  
可重入：同一线程可再次获得自己已持有的锁。

### 面试怎么答

> synchronized 互斥+可见+有序（happens-before）。  
> 可重入。实现与对象头 monitor 相关，竞争激烈会膨胀到重量级锁。  
> 锁粒度尽量小，别在锁里做远程调用。

### 和 ReentrantLock

| | synchronized | Lock |
|--|--------------|------|
| 用法 | 关键字，自动释放 | API，必须 finally unlock |
| 能力 | 够用 | tryLock、公平、多 Condition |

**完整稿：** [并发面渣](./并发面渣级口述.md)

## 四、volatile

### 白话

保证：**可见性**（改完其它线程尽快看到）和**有序性**（禁止部分重排）。  
**不保证** `i++` 这种「读-改-写」的原子性——那要用 AtomicInteger 或锁。

典型：开关标志 `running`；单例 DCL 的 `instance` 必须 volatile，防止别人看到「半初始化」对象。

### 面试怎么答

> volatile 可见+有序，不保复合操作原子。  
> i++ 不行。DCL 单例要加。

**完整稿：** [并发面渣 · volatile](./并发面渣级口述.md)

## 五、CAS

### 白话

Compare And Swap：内存值如果还是我期望的旧值，就改成新值，否则失败重试。  
乐观：假设冲突少，比悲观锁轻。AtomicInteger 底层就是它。

问题：ABA（值从 A→B→A，CAS 以为没变）→ 版本号；自旋猛会吃 CPU → LongAdder 分段。

**完整稿：** [并发面渣 · CAS](./并发面渣级口述.md)

## 六、sleep vs wait

| | sleep | wait |
|--|-------|------|
| 谁的方法 | Thread | Object |
| 锁 | **不释放** | **释放** |
| 唤醒 | 时间到 | notify / 超时 |

wait 必须在同步块里；等待条件用 **while** 判断防虚假唤醒。

## 七、P1 精讲摘要

### AQS

可以想成「同步器半成品」：一个 state + 一个排队队列。  
ReentrantLock、Semaphore、CountDownLatch 都基于它。  
面试说：**state + CLH 队列 + CAS + 模板方法** 就抓住骨架。  
**详述：** [并发面渣 · AQS](./并发面渣级口述.md)

### ThreadLocal

每个线程一份副本，互不干扰。  
实现：Thread 里的 Map，key 是弱引用。  
**坑：** 线程池线程复用，value 不 remove 会泄漏 → **finally remove**。  
**详述：** [并发面渣](./并发面渣级口述.md)

### 死锁

四条件；jstack 查；统一加锁顺序、tryLock。
