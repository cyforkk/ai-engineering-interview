## 二、P0 必会

### 2.1 线程池（第一优先级）

**参数：** corePoolSize、maximumPoolSize、keepAliveTime、workQueue、threadFactory、handler  

**流程（必须按序说）：**

```text
1. 当前 < core     → 建核心线程
2. 否则            → 入队
3. 队满且 < max    → 建非核心线程
4. 仍满            → 拒绝策略
```

| 点 | 高频结论 |
|----|----------|
| 队列 | 生产用**有界**；无界时 max 常形同虚设 |
| 拒绝 | Abort / CallerRuns / Discard / DiscardOldest |
| Executors | Fixed 易 OOM；Cached 易线程打爆 → **自建** |
| 线程数 | CPU 密≈N+1；IO 密更大；压测为准 |

**高频题**

1. 参数与执行流程？  
2. 如何设置线程数？  
3. 为何不用 Executors？  
4. 拒绝策略怎么选？  

**口述：**  
> 先核心再排队再扩容再拒绝；有界队列+明确拒绝+业务隔离池。

---

### 2.2 synchronized

- 可重入；锁对象 this / Class / 指定对象  
- 保证原子性、可见性、有序性（hb）  
- 与 Lock：API 锁可 tryLock、公平、多 Condition，须 finally unlock  

**高频题：** synchronized 原理？与 ReentrantLock 区别？

---

### 2.3 volatile

| 能 | 不能 |
|----|------|
| 可见性 | i++ 原子性 |
| 禁止部分重排 | 替代互斥锁 |

**场景：** 标志位、DCL 单例的 `instance`。  

**高频题：** volatile 作用？能保证 i++ 吗？DCL 为何要 volatile？

---

### 2.4 CAS

- Compare And Swap；AtomicInteger  
- ABA → 版本号；自旋猛 → LongAdder  

**高频题：** CAS 是什么？ABA？缺点？

---

### 2.5 线程基础

| 点 | 结论 |
|----|------|
| 创建 | Thread / Runnable / Callable；生产用池 |
| 状态 | NEW→RUNNABLE→BLOCKED/WAITING/TIMED_WAITING→TERMINATED |
| sleep | 不释放锁 |
| wait | 释放锁，需同步块 + notify |

**高频题：** start vs run？sleep vs wait？

## 三、P1 常考

### 3.1 AQS

- `state` + CLH 变体队列  
- 独占（Lock）/ 共享（Semaphore、Latch）  
- ReentrantLock 基石  

**高频题：** AQS 是什么？独占共享区别？

### 3.2 ReentrantLock

可公平、tryLock、可中断、多 Condition；finally unlock。

### 3.3 ThreadLocal

- Thread 内 ThreadLocalMap；key 弱引用  
- **线程池必须 finally remove**  

**高频题：** 原理？为何泄漏？

### 3.4 死锁

四条件；jstack 查；固定加锁顺序、tryLock。

### 3.5 ConcurrentHashMap（并发视角）

CAS + synchronized 桶头；禁 null；get 基本无锁。  
（结构细节见 Java 专题）

### 3.6 单例并发

DCL + volatile；静态内部类；枚举。
