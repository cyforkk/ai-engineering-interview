## 1. 虚拟线程（Virtual Thread）

**是什么：** JDK 21 正式的轻量用户态线程，由 JVM 调度，适合 **大量阻塞 IO** 并发，而不是算力并行。

**口述 40 秒：**

> 传统平台线程贵、创建数量受限；虚拟线程很便宜，阻塞时会卸载载体线程，适合高并发阻塞式服务。  
> 它不替代认真设计的线程池业务隔离；CPU 密集仍要控制并行度。  
> 注意：同步阻塞在 `synchronized` 上可能钉住载体线程；尽量用 `ReentrantLock` 等，库要虚拟线程友好。

**追问：** 和 WebFlux/响应式比？→ 同步写法扩并发，心智简单；响应式仍适复杂背压与流式。  
**链：** [并发完整卷](./并发高频面试题与知识点.md)

## 2. 结构化并发 / ScopedValue（了解级）

**是什么：** 把「一组子任务」当一个单元取消/错误传播；`ScopedValue` 替代部分 ThreadLocal 传递（虚拟线程下更友好）。

**口述：** 面试提到「虚拟线程时代如何管生命周期」——结构化并发避免线程泄漏；细节可不背 API，要说清 **问题域**。

## 3. Spring Boot 3 / Spring Framework 6 差异（常问）

| 点 | 怎么说 |
|----|--------|
| 基线 | Boot 3 需要 **Java 17+** |
| `javax.*` → `jakarta.*` | 包名迁移是升级最大坑 |
| 原生/AOT | GraalVM 原生镜像、AOT 是加分了解 |
| 观测 | Micrometer  tracing 等更一等公民（会说链路即可） |

**口述：** 我升级/新建项目会默认 Boot 3 + JDK 17/21；迁移先盯 jakarta 与依赖兼容，再谈特性。

**链：** [Spring 完整卷](./Spring高频面试题与知识点.md)

## 4. 线上指标你会看什么（比背名词加分）

**口述：**

> 接口：QPS、P95/P99、错误率、饱和度（线程池队列、DB 连接池）。  
> JVM：GC 停顿、堆使用、线程数、老年代晋升。  
> 依赖：下游 RT、超时、熔断次数。  
> 变更：发布前后对比，先止血再根因。

**链：** [场景题·排查](./场景题高频面试题与知识点.md) · [JVM 面渣](./JVM面渣级口述.md)

## 5. 记录类 / 模式匹配 / 密封类（语言加分，短说）

- `record`：不可变数据载体，少样板代码  
- `switch` 模式匹配：少强转  
- `sealed`：限制继承层级，领域建模更严  

**原则：** 能举 1 个业务 DTO 用 record 即可，别展开语法课。
